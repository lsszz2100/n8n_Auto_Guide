# 레슨 2.6: n8n 인터페이스 완전 정복

---

## n8n 화면 구성 개요

n8n 인터페이스는 크게 세 가지 영역으로 구성됩니다:

```
┌─────────────────────────────────────────────────┐
│  상단 내비게이션 바                               │
├──────────┬──────────────────────────────────────┤
│          │                                      │
│  사이드바  │           메인 캔버스                │
│  (좌측)   │        (워크플로우 편집 영역)          │
│          │                                      │
└──────────┴──────────────────────────────────────┘
```

---

## 1. 대시보드 (홈 화면)

로그인하면 가장 먼저 보이는 화면입니다.

### 주요 요소

**최근 워크플로우**: 마지막으로 수정한 워크플로우 목록

**실행 현황**: 최근 실행된 워크플로우의 성공/실패 통계

**퀵 액션**:
- `+ New Workflow`: 새 워크플로우 생성
- `Import`: JSON 파일로 워크플로우 가져오기

---

## 2. 좌측 사이드바 메뉴

| 아이콘 | 메뉴 | 설명 |
|--------|------|------|
| 🏠 | Home | 대시보드 홈 |
| ⚡ | Workflows | 전체 워크플로우 목록 |
| 🔑 | Credentials | API 키, 비밀번호 등 인증 정보 관리 |
| 📋 | Executions | 실행 기록 로그 |
| 🔧 | Variables | 전역 변수 관리 (Enterprise) |
| 👥 | Users | 팀원 관리 (Cloud/Enterprise) |
| ⚙️ | Settings | 시스템 설정 |

---

## 3. 워크플로우 편집기 (핵심!)

워크플로우를 만드는 메인 화면입니다. 가장 많은 시간을 보내게 될 공간입니다.

### 편집기 상단 도구모음

```
[워크플로우 이름] [저장] [비활성/활성]  [실행] [...]
```

- **워크플로우 이름**: 클릭하여 이름 변경
- **저장(Ctrl+S)**: 변경사항 저장
- **토글 스위치**: 워크플로우 활성화/비활성화
- **Execute Workflow**: 수동으로 워크플로우 즉시 실행
- **...** (더보기): 내보내기, 복제, 삭제 등

### 캔버스 조작법

| 동작 | 방법 |
|------|------|
| 이동 | 마우스 드래그 (빈 공간에서) |
| 확대/축소 | Ctrl + 마우스 휠 |
| 화면 맞춤 | Ctrl + Shift + H |
| 노드 선택 | 클릭 |
| 다중 선택 | Shift + 클릭 |
| 노드 이동 | 선택 후 드래그 |
| 노드 복사 | Ctrl + C / Ctrl + V |
| 노드 삭제 | Delete 키 |
| 되돌리기 | Ctrl + Z |

---

## 4. 노드 추가하기

### 방법 1: 캔버스에서 "+" 버튼 클릭
노드의 연결 점(오른쪽 끝)에서 "+" 버튼을 클릭하면 노드 선택 패널이 열립니다.

### 방법 2: 캔버스 빈 곳에서 더블클릭
빈 캔버스를 더블클릭하면 노드 검색 패널이 열립니다.

### 방법 3: Tab 키
편집기에서 Tab 키를 누르면 노드 검색 패널이 열립니다.

### 노드 검색 패널 구성
```
┌─────────────────────────────┐
│ 🔍 검색창                   │
├─────────────────────────────┤
│ ⭐ 즐겨찾기                  │
│ 🔄 최근 사용                │
│ 📂 카테고리별 목록           │
│   - Triggers (트리거)        │
│   - Core Nodes (핵심 노드)  │
│   - Actions in an App       │
└─────────────────────────────┘
```

---

## 5. 노드 설정 패널

노드를 클릭하면 우측에 설정 패널이 열립니다.

### 패널 구성

**Parameters 탭 (기본)**:
- 노드별 설정 항목
- 입력 필드, 드롭다운, 토글 등

**Settings 탭**:
- 노드 이름 변경
- 에러 처리 방식
- 메모 작성
- 실행 대기(Retry) 설정

**Notes 탭**:
- 노드에 대한 메모 작성
- 팀원들을 위한 설명 추가 가능

---

## 6. 데이터 패널 (실행 후)

워크플로우를 실행하면 각 노드 아래에 결과가 표시됩니다.

### 입력/출력 데이터 보기
- 노드 클릭 → 패널 상단에 Input / Output 탭
- 각 아이템의 JSON 데이터 확인 가능
- 특정 아이템을 선택하여 상세 보기

### 데이터 패널 활용 팁
1. **스키마 보기**: 데이터 구조를 트리 형태로 확인
2. **테이블 보기**: 표 형식으로 여러 아이템 비교
3. **표현식 복사**: 필드명을 클릭하면 표현식이 자동 복사

---

## 7. 크리덴셜 관리

크리덴셜(Credentials)은 외부 서비스 연결을 위한 인증 정보입니다.

### 크리덴셜 추가하기
1. 사이드바 → Credentials → "+ Add credential"
2. 서비스 선택 (예: Google, Slack)
3. 인증 방법 선택 (OAuth2 또는 API Key)
4. 인증 정보 입력 및 저장

### 크리덴셜 보안
- 저장된 크리덴셜은 암호화됨
- 다른 팀원과 공유 가능 (권한 설정 필요)
- 크리덴셜 자체를 내보내면 복호화되므로 주의

---

## 8. 실행 기록 (Executions)

사이드바 → Executions에서 모든 실행 기록을 확인할 수 있습니다.

### 실행 기록에서 확인 가능한 것
- 실행 시작/완료 시간
- 성공/실패/대기 상태
- 각 노드의 입출력 데이터
- 에러 메시지와 스택 트레이스

### 특정 실행 재시도
실패한 실행을 클릭 → "Retry" 버튼으로 동일한 데이터로 재실행 가능
