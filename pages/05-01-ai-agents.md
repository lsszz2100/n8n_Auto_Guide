# 레슨 5.1: AI 에이전트 완전 이해

---

## AI 에이전트란 무엇인가?

AI 에이전트(AI Agent)는 단순히 AI를 호출하는 것과 다릅니다. 에이전트는 **목표를 주면 스스로 계획을 세우고, 도구를 사용하며, 결과를 달성**합니다.

```
일반 AI 호출:
  입력 → AI → 출력
  (한 번의 요청, 한 번의 응답)

AI 에이전트:
  목표 → [계획 수립] → [도구 선택] → [실행] → [결과 확인] → [재시도/완료]
          (여러 단계, 스스로 결정)
```

### 실생활 비유

비서에게 "내일 회의 준비해줘"라고 하면:
- 일반 자동화: 미리 정의된 회의 준비 체크리스트만 실행
- AI 에이전트: 캘린더 확인 → 참석자 조회 → 관련 문서 수집 → 안건 초안 작성 → 참석자에게 초대장 발송

AI 에이전트는 **맥락을 이해하고 자율적으로 행동**합니다.

---

## AI 에이전트 vs 일반 자동화

| 비교 항목 | 일반 자동화 | AI 에이전트 |
|----------|-----------|------------|
| 판단 기준 | 미리 정의된 규칙 | 상황에 따른 추론 |
| 예외 처리 | 정의된 케이스만 | 새로운 상황 대응 가능 |
| 유연성 | 낮음 (규칙 변경 필요) | 높음 (프롬프트 조정) |
| 비용 | 낮음 | 높음 (API 비용) |
| 예측 가능성 | 높음 | 중간 (결과가 다양) |
| 복잡한 언어 처리 | 불가능 | 가능 |
| 학습/개선 | 수동 업데이트 | 프롬프트 개선으로 가능 |

**언제 AI 에이전트를 사용하나?**
- 비정형 텍스트 처리 (이메일, 메시지, 문서)
- 복잡한 판단이 필요한 경우
- 여러 시스템을 조율해야 할 때
- 예외 케이스가 너무 많아 규칙화가 어려울 때

**일반 자동화가 더 나은 경우:**
- 단순하고 명확한 규칙 기반 작업
- 비용이 중요한 경우
- 100% 일관된 결과가 필요한 경우

## 4가지 에이전트 유형

### 1. Reactive Agent (반응형 에이전트)

가장 단순한 형태. 입력에 즉각 반응합니다.

```
특징: 현재 상태만 보고 반응
예시: 챗봇 FAQ, 감정 분석, 즉각적인 분류

동작 방식:
  입력(현재 메시지) → AI 판단 → 즉각 응답

n8n 활용:
  [Webhook] → [Basic LLM Chain] → [응답 반환]
```

**장점**: 빠르고 단순
**단점**: 이전 대화나 맥락을 고려하지 않음

---

### 2. Goal-based Agent (목표 기반 에이전트)

목표를 달성하기 위해 여러 단계로 계획을 세웁니다.

```
특징: 목표 설정 → 계획 → 실행 → 목표 달성 확인
예시: 여행 예약, 리서치 에이전트, 데이터 수집 자동화

동작 방식:
  목표 설정 → [계획 수립] → [단계별 실행] → [결과 검증] → 완료/재시도

n8n 활용:
  [AI Agent 노드] + [검색 도구] + [이메일 도구] + [캘린더 도구]
```

**장점**: 복잡한 다단계 작업 처리 가능
**단점**: 실행 시간이 길고 비용 증가

---

### 3. Utility-based Agent (효용 기반 에이전트)

여러 옵션 중 가장 좋은 결과를 선택합니다.

```
특징: 각 선택지의 가치를 평가하고 최적안 선택
예시: 최적 공급업체 선택, 리소스 배분, 일정 최적화

동작 방식:
  여러 옵션 생성 → [각 옵션 평가] → [최적 옵션 선택] → 실행

n8n 활용:
  [AI Agent] → 여러 API 호출 → [결과 비교] → [최적 결과 반환]
```

**장점**: 최적화된 결정
**단점**: 설계가 복잡

---

### 4. Learning Agent (학습형 에이전트)

피드백을 통해 점점 더 나아지는 에이전트입니다.

```
특징: 결과 피드백 → 행동 개선 반복
예시: 추천 시스템, 개인화된 콘텐츠, 점진적으로 개선되는 챗봇

동작 방식:
  실행 → [피드백 수집] → [패턴 학습] → [행동 개선] → 더 나은 실행

n8n 활용:
  [AI Agent] + [Google Sheets에 피드백 저장] + [Few-shot 예시 누적]
```

**장점**: 시간이 지날수록 성능 향상
**단점**: 초기 구축이 가장 복잡

## AI 에이전트가 빛나는 곳

### 사례 1: 고객 지원 자동화

```
기존 방식:
  고객 메시지 → 키워드 감지 → 정해진 답변 or 사람에게 전달

AI 에이전트:
  고객 메시지 → [의도 파악] → [주문 DB 조회] → [FAQ 검색] → [맞춤 답변 생성]

결과: 80%의 문의를 자동 처리, 복잡한 건만 사람에게 전달
```

### 사례 2: 콘텐츠 생성 파이프라인

```
AI 에이전트 역할:
  트렌드 키워드 수집 → [관련 정보 리서치] → [초안 작성] → [SEO 최적화] → [발행]

결과: 하루 1개 → 하루 10개 콘텐츠 발행 가능
```

### 사례 3: 데이터 분석 보고서

```
AI 에이전트 역할:
  데이터 파일 수신 → [데이터 분석] → [인사이트 도출] → [보고서 작성] → [슬랙 공유]

결과: 2시간 걸리던 주간 보고서가 5분으로 단축
```

### 사례 4: 영업 리드 처리

```
AI 에이전트 역할:
  리드 폼 수신 → [회사 정보 조사] → [리드 스코어 계산] → [맞춤 이메일 작성] → [CRM 등록]

결과: 리드 처리 시간 90% 단축
```

## n8n에서 AI 에이전트의 위치

n8n은 AI 에이전트를 위한 이상적인 플랫폼입니다.

```
n8n AI 에이전트 구조:

[트리거]
    ↓
[AI Agent 노드]
  ├── Brain (LLM): OpenAI / Anthropic / Google
  ├── Tools: 이메일, DB, 검색, 계산기, HTTP 요청 등
  └── Memory: 대화 기록, 벡터 스토어
    ↓
[결과 처리 노드들]
    ↓
[최종 액션]
```

### n8n이 AI 에이전트에 특별한 이유

1. **500+ 통합**: AI가 사용할 수 있는 도구가 이미 준비됨
2. **시각적 구성**: 코드 없이 에이전트 파이프라인 설계
3. **하이브리드**: 규칙 기반 + AI 기반을 자유롭게 혼합
4. **Self-hosted**: 민감한 데이터도 안전하게 처리
5. **디버깅 용이**: 각 단계별 실행 결과 확인 가능

## ReAct 패턴 이해하기

n8n의 AI Agent 노드는 **ReAct(Reasoning + Acting)** 패턴을 사용합니다.

```
ReAct 사이클:

1. Thought (생각): "이 문의는 배송 관련이다. 주문 번호가 필요하다."
2. Action (행동): 주문 DB 조회 도구 실행
3. Observation (관찰): "주문 #12345, 2024-01-15 배송 출발"
4. Thought: "배송이 출발했으니 추적 링크를 포함해 답변해야겠다."
5. Action: 고객에게 답변 이메일 발송
6. Final Answer: 완료
```

이 사이클이 목표를 달성할 때까지 반복됩니다. n8n에서는 이 과정이 내부적으로 자동 관리됩니다.

## 실전 예제: 첫 번째 AI 에이전트 만들기

이메일 문의를 자동 분류하고 답변하는 간단한 AI 에이전트를 처음부터 구축합니다.

### 목표

```
"고객 이메일이 오면 AI가 내용을 파악하여,
주문/배송/환불/일반 문의 중 하나로 분류하고
각 유형에 맞는 답변을 자동으로 생성한다."
```

### 전체 워크플로우

```
[Gmail Trigger: 새 이메일]
    ↓
[AI Agent 노드]
  ├── LLM: OpenAI gpt-4o-mini
  ├── System Prompt: 고객지원 에이전트 역할 부여
  ├── Tool 1: 주문 조회 (HTTP Request to DB API)
  └── Tool 2: FAQ 검색 (Vector Store)
    ↓
[Gmail: AI가 생성한 답변 발송]
```

### AI Agent 노드 설정 방법

```
1. n8n 캔버스 → 노드 추가 → "AI Agent" 검색

2. Chat Model 탭:
   - 모델: OpenAI Chat Model
   - 크리덴셜: OpenAI API 키
   - Model: gpt-4o-mini (비용 절약)

3. System Message 설정:
   당신은 [회사명]의 고객지원 AI입니다.
   고객 문의를 친절하게 처리하고, 필요하면 제공된 도구를 사용하세요.
   
   분류 기준:
   - order: 주문, 결제 관련 문의
   - shipping: 배송, 추적 관련 문의
   - refund: 환불, 교환 관련 문의
   - general: 기타 일반 문의
   
   항상 한국어로 답변하세요.

4. Tools 탭 → Tool 추가:
   도구 이름: search_order
   도구 설명: 주문 번호로 주문 상태를 조회합니다
   → HTTP Request 노드 연결

5. Memory 탭:
   - Window Buffer Memory 추가
   - Context Window: 10 (최근 10개 메시지 기억)
```

### System Prompt 심화 작성 예시

```
당신은 [쇼핑몰명] 고객지원 전문 AI입니다.

## 역할
- 고객 문의를 정확히 이해하고 신속하게 처리
- 필요시 도구를 사용하여 정확한 정보 제공
- 해결 불가한 경우 담당자 연결 안내

## 답변 형식
1. 문의 내용 요약 (1줄)
2. 확인된 정보 또는 조치 내용
3. 추가 도움 필요 여부 확인

## 제약사항
- 개인정보(주민번호, 카드번호 등) 절대 요청 금지
- 환불 승인은 직접 하지 말고 처리 절차 안내만
- 확실하지 않은 정보는 "확인 후 연락드리겠습니다" 답변
```

## AI 에이전트 도구(Tool) 추가 방법

도구는 AI가 직접 호출할 수 있는 외부 기능입니다. n8n에서 모든 노드를 도구로 등록할 수 있습니다.

### 도구 유형별 활용

| 도구 유형 | 용도 | n8n 노드 |
|----------|------|----------|
| HTTP Request | 외부 API 조회 | HTTP Request Tool |
| 데이터베이스 조회 | 주문/고객 정보 | MySQL, PostgreSQL Tool |
| 벡터 검색 | FAQ, 문서 검색 | Vector Store Tool |
| 코드 실행 | 계산, 데이터 처리 | Code Tool |
| 계산기 | 숫자 연산 | Calculator Tool |
| 날짜/시간 | 날짜 계산 | DateTime Tool |

### 도구 설명 잘 쓰기

AI는 도구 설명을 보고 언제 사용할지 결정합니다. 명확하게 작성해야 합니다.

```
나쁜 도구 설명:
"데이터를 가져온다"

좋은 도구 설명:
"고객의 주문 번호(예: ORD-12345)를 입력하면 주문 상태, 배송 정보,
예상 도착일을 반환합니다. 주문 조회가 필요할 때 사용하세요."
```

## 에이전트 성능 개선 팁

### 1. 프롬프트 반복 개선

```
테스트 → 결과 확인 → 프롬프트 수정 → 재테스트

기록할 것:
- 어떤 입력에서 실패했나?
- AI가 잘못 이해한 부분은?
- 어떤 도구를 엉뚱하게 사용했나?
```

### 2. Few-shot 예시 추가

```
System Prompt에 예시 추가:
예시 1:
  입력: "어제 주문했는데 아직 배송이 안 됐어요"
  처리: search_order 도구 → 주문 조회 → 배송 상태 답변
  분류: shipping

예시 2:
  입력: "영수증 재발급 가능한가요?"
  처리: FAQ 검색 → 영수증 재발급 절차 안내
  분류: general
```

### 3. 출력 파싱

에이전트의 출력을 구조화하여 다음 노드에서 활용:

```javascript
// Code 노드: 에이전트 응답 파싱
const agentResponse = $json.output;

// 분류 추출 (에이전트가 JSON 형식으로 답변하도록 프롬프트 설계)
let parsed;
try {
  parsed = JSON.parse(agentResponse);
} catch {
  parsed = { category: 'general', answer: agentResponse };
}

return [{ json: parsed }];
```

## 핵심 요약

- AI 에이전트는 목표를 주면 스스로 계획하고 도구를 사용해 달성합니다
- 4가지 유형: Reactive(반응형), Goal-based(목표 기반), Utility-based(효용 기반), Learning(학습형)
- 비정형 데이터 처리, 복잡한 판단, 다중 시스템 조율에 가장 적합합니다
- n8n은 500+ 통합을 AI의 도구로 활용할 수 있는 최적의 플랫폼입니다
- ReAct 패턴으로 생각 → 행동 → 관찰 사이클을 반복합니다
- 도구 설명을 명확하게 작성해야 AI가 올바른 도구를 선택합니다
- 프롬프트는 테스트 → 개선 → 재테스트 사이클로 반복 개선합니다
