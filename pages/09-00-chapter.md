# 9장: n8n 운영 및 관리

---

## 이 챕터에서 배울 것

자동화 시스템은 만드는 것보다 **운영하는 것**이 더 중요합니다. 안정적으로 운영하고, 팀과 협력하며, 문제 발생 시 빠르게 대응하는 방법을 다룹니다.

## 왜 운영이 중요한가?

### 자동화가 멈추면 발생하는 일

```
예시: 이커머스 자동화 장애 시나리오

오전 2시: n8n 서버 디스크 꽉 참
오전 2:01: 주문 처리 워크플로우 중단
오전 9시: 직원 출근 후 이상 발견
   → 7시간 동안 100건의 주문 미처리
   → 재고 부족 상품 계속 판매됨
   → 고객 불만 급증
오전 10시: 수동으로 미처리 주문 처리 시작
오후 3시: 모든 주문 처리 완료

결과: 5시간의 수동 작업 + 고객 신뢰 손상
```

**모니터링이 있었다면:** 오전 2시에 Slack 알림, 즉시 대응 → 피해 없음

## 이 챕터의 목표

- 워크플로우와 데이터를 정기적으로 **백업**
- n8n 버전을 안전하게 **업그레이드**
- 팀원들과 효율적으로 **협업**
- 문제 발생 전에 미리 **감지**
