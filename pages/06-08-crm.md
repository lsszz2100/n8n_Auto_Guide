# 레슨 6.8: CRM 영업 자동화

---

## CRM 자동화가 필요한 이유

영업팀이 CRM에 데이터를 수동으로 입력하는 데 쏟는 시간: **하루 평균 2시간 이상**

이 시간을 자동화로 0으로 만들 수 있습니다.

---

## 지원 CRM 도구

n8n에서 네이티브 지원하는 주요 CRM:
- **HubSpot**: 무료 플랜 있음, 가장 인기
- **Salesforce**: 대기업 표준
- **Pipedrive**: 영업 중심 CRM
- **Zoho CRM**: 합리적 가격
- **Airtable**: 유연한 커스텀 CRM

### 예제 1: 신규 리드 스코어링 자동화 (회사 규모, 직책, 행동 데이터 기반 점수화)

새 리드가 들어올 때 회사 정보, 직책 시니어리티, 웹사이트 행동 패턴(페이지 방문수, 자료 다운로드 등)을 조합하여 수치 기반 점수를 산출합니다. 고득점 리드는 즉시 시니어 영업담당자에게 배정됩니다.

워크플로우 구조:

```
[Webhook: /crm/new-lead]
  새 리드 데이터 수신
    ↓
[HTTP Request: Clearbit API - 회사 정보 보강]
  회사 규모, 업종, 연간 매출 조회
    ↓
[Code: 규칙 기반 1차 점수 계산]
  회사 규모 + 직책 + 소스 점수 합산
    ↓
[HTTP Request: 내부 행동 데이터 API - 사이트 활동 조회]
  방문 횟수, 열람 페이지, 자료 다운로드 이력
    ↓
[Code: 행동 점수 추가 + 최종 등급 결정]
    ↓
[HubSpot: Create Contact + 점수/등급 프로퍼티 저장]
    ↓
[Switch: 리드 등급]
  ├── S (90점 이상)  → [HubSpot: 시니어 영업 배정] + [Slack: 즉시 알림]
  ├── A (70-89점)   → [HubSpot: 중간 영업 배정] + [이메일: 당일 연락 안내]
  ├── B (50-69점)   → [HubSpot: 주니어 배정] + [이메일 시퀀스 등록]
  └── C (49점 이하) → [HubSpot: 자동 육성 시퀀스 등록]
```

Code 노드 - 규칙 기반 1차 점수 계산:

```javascript
const lead = $input.item.json.leadData;
const company = $input.item.json.clearbitData ?? {};

let score = 0;
const scoreBreakdown = [];

// 1. 회사 규모 점수 (최대 30점)
const employeeCount = company.metrics?.employees ?? 0;
let sizeScore = 0;
if (employeeCount >= 1000) sizeScore = 30;
else if (employeeCount >= 200) sizeScore = 22;
else if (employeeCount >= 50) sizeScore = 15;
else if (employeeCount >= 10) sizeScore = 8;
else sizeScore = 3;
score += sizeScore;
scoreBreakdown.push(`회사 규모 (${employeeCount}명): +${sizeScore}`);

// 2. 직책 시니어리티 점수 (최대 25점)
const title = (lead.jobTitle ?? '').toLowerCase();
let titleScore = 0;
if (/ceo|cto|coo|founder|대표|부대표/.test(title)) titleScore = 25;
else if (/vp|director|본부장|이사|부장/.test(title)) titleScore = 18;
else if (/manager|팀장|리드/.test(title)) titleScore = 12;
else if (/senior|시니어/.test(title)) titleScore = 8;
else titleScore = 4;
score += titleScore;
scoreBreakdown.push(`직책 (${lead.jobTitle}): +${titleScore}`);

// 3. 업종 적합성 점수 (최대 20점)
const industry = (company.category?.industry ?? '').toLowerCase();
const targetIndustries = ['software', 'technology', 'saas', 'e-commerce', 'fintech'];
const industryScore = targetIndustries.some(i => industry.includes(i)) ? 20 : 8;
score += industryScore;
scoreBreakdown.push(`업종 적합성: +${industryScore}`);

// 4. 리드 소스 점수 (최대 15점)
const sourceScoreMap = {
  '추천': 15,
  'LinkedIn': 12,
  '웹사이트': 10,
  '이벤트': 10,
  '콜드아웃바운드': 4
};
const sourceScore = sourceScoreMap[lead.source] ?? 5;
score += sourceScore;
scoreBreakdown.push(`소스 (${lead.source}): +${sourceScore}`);

return [{
  json: {
    ...lead,
    companyInfo: company,
    phase1Score: score,
    scoreBreakdown,
    employeeCount
  }
}];
```

Code 노드 - 행동 점수 추가 및 최종 등급 결정:

```javascript
const data = $input.item.json;
const activity = $('HTTP Request: 행동 데이터').item.json ?? {};

let behaviorScore = 0;
const behaviorBreakdown = [];

// 페이지 방문 횟수 (최대 20점)
const visitCount = activity.visitCount ?? 0;
const visitScore = Math.min(visitCount * 2, 20);
behaviorScore += visitScore;
behaviorBreakdown.push(`방문 횟수 (${visitCount}회): +${visitScore}`);

// 핵심 페이지 방문 여부 (최대 15점)
const keyPages = activity.visitedPages ?? [];
const keyPageScore = keyPages.includes('pricing') ? 15
  : keyPages.includes('features') ? 8
  : 0;
behaviorScore += keyPageScore;
behaviorBreakdown.push(`핵심 페이지 방문: +${keyPageScore}`);

// 자료 다운로드 횟수 (최대 15점)
const downloadCount = activity.downloadCount ?? 0;
const downloadScore = Math.min(downloadCount * 5, 15);
behaviorScore += downloadScore;
behaviorBreakdown.push(`자료 다운로드 (${downloadCount}건): +${downloadScore}`);

const totalScore = data.phase1Score + behaviorScore;

// 등급 결정
let grade;
if (totalScore >= 90) grade = 'S';
else if (totalScore >= 70) grade = 'A';
else if (totalScore >= 50) grade = 'B';
else grade = 'C';

return [{
  json: {
    ...data,
    behaviorScore,
    totalScore,
    grade,
    allBreakdown: [...data.scoreBreakdown, ...behaviorBreakdown]
  }
}];
```

### 예제 2: 고객 이탈 예측 + 재활성화 캠페인 자동 트리거

마지막 로그인 일수, 최근 구매 일수, 고객 지원 문의 이력을 기반으로 이탈 위험 고객을 매일 자동 감지하고, 위험 수준별로 차별화된 재활성화 캠페인을 트리거합니다.

워크플로우 구조:

```
[Schedule: 매일 오전 7시 (0 7 * * *)]
    ↓
[HTTP Request: 내부 DB API - 최근 30일 비활성 고객 목록 조회]
    ↓
[Code: 이탈 위험 점수 계산]
  마지막 활동일, 구매 주기, 지원 문의 패턴 분석
    ↓
[Split In Batches: 고객별 처리]
    ↓
[HubSpot: 고객 정보 + 구매 이력 조회]
    ↓
[Code: 최종 이탈 위험 등급 결정]
    ↓
[Switch: 위험 등급]
  ├── 위험 (30일 이상 비활성)
  │     → [HubSpot: 이탈위험 태그 추가]
  │     → [Gmail: 특별 할인 쿠폰 이메일]
  │     → [Slack: 담당 영업에게 알림]
  │
  ├── 주의 (15-29일 비활성)
  │     → [HubSpot: 관심유도 시퀀스 등록]
  │     → [Gmail: 새 기능 소개 이메일]
  │
  └── 관찰 (7-14일 비활성)
        → [Gmail: 뉴스레터/유용한 콘텐츠 발송]
```

Code 노드 - 이탈 위험 점수 계산:

```javascript
const customers = $input.all();
const today = new Date();

const analyzed = customers.map(item => {
  const customer = item.json;

  // 마지막 로그인 경과일
  const lastLogin = new Date(customer.lastLoginAt ?? customer.createdAt);
  const daysSinceLogin = Math.floor((today - lastLogin) / (1000 * 60 * 60 * 24));

  // 마지막 구매 경과일
  const lastPurchase = customer.lastPurchaseAt ? new Date(customer.lastPurchaseAt) : null;
  const daysSincePurchase = lastPurchase
    ? Math.floor((today - lastPurchase) / (1000 * 60 * 60 * 24))
    : 999;

  // 이탈 위험 점수 (높을수록 위험)
  let churnScore = 0;

  // 로그인 비활성 기간
  if (daysSinceLogin >= 30) churnScore += 40;
  else if (daysSinceLogin >= 15) churnScore += 25;
  else if (daysSinceLogin >= 7) churnScore += 10;

  // 구매 비활성 기간 (유료 고객만)
  if (customer.isPaid && daysSincePurchase >= 90) churnScore += 35;
  else if (customer.isPaid && daysSincePurchase >= 45) churnScore += 20;

  // 고객 지원 부정 이력 (미해결 문의 존재 시)
  if (customer.hasUnresolvedTicket) churnScore += 15;

  // 플랜 다운그레이드 이력
  if (customer.hasDowngraded) churnScore += 10;

  // 위험 등급
  let riskLevel;
  if (churnScore >= 55) riskLevel = '위험';
  else if (churnScore >= 30) riskLevel = '주의';
  else riskLevel = '관찰';

  return {
    customerId: customer.id,
    customerName: customer.name,
    email: customer.email,
    hubspotContactId: customer.hubspotContactId,
    assigneeSlackId: customer.assigneeSlackId,
    plan: customer.plan,
    isPaid: customer.isPaid,
    daysSinceLogin,
    daysSincePurchase,
    churnScore,
    riskLevel,
    hasUnresolvedTicket: customer.hasUnresolvedTicket
  };
});

// 이탈 위험 점수 높은 순 정렬
analyzed.sort((a, b) => b.churnScore - a.churnScore);

return analyzed.map(c => ({ json: c }));
```

```javascript
  `재활성화 할인 이메일은 자동 발송되었습니다.`,
  `추가 개인 연락을 권장합니다.`
].join('\n');

return [{ json: { message, channel } }];
```

### 예제 3: 영업 파이프라인 주간 리포트 자동 생성 + 슬랙 발송

매주 금요일 오후 HubSpot의 파이프라인 데이터를 종합하여 팀 성과 리포트를 자동 생성합니다. 스테이지별 딜 현황, 이번 주 신규/성사/실패 건수, 다음 주 예상 클로징 딜을 포함합니다.

워크플로우 구조:

```
[Schedule: 매주 금요일 오후 5시 (0 17 * * 5)]
    ↓
병렬 데이터 수집:
  ├── [HubSpot: 이번 주 신규 Deal 조회]
  ├── [HubSpot: 이번 주 클로즈된 Deal 조회 (Won/Lost)]
  ├── [HubSpot: 다음 주 클로징 예정 Deal 조회]
  └── [HubSpot: 전체 파이프라인 스테이지별 현황]
    ↓
[Merge: 4개 데이터 합산]
    ↓
[Code: 주간 성과 지표 계산]
  신규 리드수, 성사율, 총 파이프라인 금액 등
    ↓
[OpenAI: 주간 성과 인사이트 + 다음 주 권장 액션 생성]
    ↓
[Code: Slack 블록킷 메시지 조립]
    ↓
[Slack: #sales 채널에 주간 리포트 발송]
    ↓
[Notion: 주간 리포트 페이지 자동 생성]
```

HubSpot 이번 주 신규 Deal 조회 노드:

```
[HubSpot: Search Deals]
  Filter:
    createdate: BETWEEN {{ $now.startOf('week').toISO() }} AND {{ $now.endOf('week').toISO() }}
  Properties: dealname, amount, dealstage, hubspot_owner_id, closedate, pipeline
  Limit: 200
```

HubSpot 클로즈된 Deal 조회 노드:

```
[HubSpot: Search Deals]
  Filter:
    closedate: BETWEEN {{ $now.startOf('week').toISO() }} AND {{ $now.endOf('week').toISO() }}
    dealstage: IN [closedwon, closedlost]
  Properties: dealname, amount, dealstage, hubspot_owner_id, closedate
  Limit: 200
```

Code 노드 - 주간 성과 지표 계산:

```javascript
// 각 HubSpot 조회 결과 수집
const newDeals = $('HubSpot: 신규 Deal').all().map(i => i.json);
const closedDeals = $('HubSpot: 클로즈 Deal').all().map(i => i.json);
const upcomingDeals = $('HubSpot: 다음 주 클로징').all().map(i => i.json);
const pipelineAll = $('HubSpot: 전체 파이프라인').all().map(i => i.json);

// 이번 주 성사/실패 분류
const wonDeals = closedDeals.filter(d => d.dealstage === 'closedwon');
const lostDeals = closedDeals.filter(d => d.dealstage === 'closedlost');

// 금액 합산
const wonAmount = wonDeals.reduce((sum, d) => sum + (Number(d.amount) || 0), 0);
const lostAmount = lostDeals.reduce((sum, d) => sum + (Number(d.amount) || 0), 0);
const newPipelineAmount = newDeals.reduce((sum, d) => sum + (Number(d.amount) || 0), 0);
const upcomingAmount = upcomingDeals.reduce((sum, d) => sum + (Number(d.amount) || 0), 0);

// 전체 파이프라인 금액 (오픈 딜)
const totalPipelineAmount = pipelineAll
  .filter(d => !['closedwon', 'closedlost'].includes(d.dealstage))
  .reduce((sum, d) => sum + (Number(d.amount) || 0), 0);

// 성사율
const totalClosed = wonDeals.length + lostDeals.length;
const winRate = totalClosed > 0 ? Math.round((wonDeals.length / totalClosed) * 100) : 0;

// 담당자별 성과
const byOwner = {};
for (const deal of wonDeals) {
  const owner = deal.hubspot_owner_id ?? '미지정';
  byOwner[owner] = (byOwner[owner] ?? 0) + (Number(deal.amount) || 0);
}
const topPerformer = Object.entries(byOwner)
  .sort((a, b) => b[1] - a[1])[0];

const weekStr = `${new Date().toISOString().slice(0, 10)} 주`;

return [{
  json: {
    weekStr,
    newDealsCount: newDeals.length,
    newPipelineAmount,
    wonDealsCount: wonDeals.length,
    wonAmount,
    lostDealsCount: lostDeals.length,
    lostAmount,
    winRate,
    upcomingDealsCount: upcomingDeals.length,
    upcomingAmount,
    totalPipelineAmount,
    topPerformerOwnerId: topPerformer?.[0] ?? '없음',
    topPerformerAmount: topPerformer?.[1] ?? 0,
    wonDealsList: wonDeals.map(d => `${d.dealname} (${Number(d.amount || 0).toLocaleString()}원)`).join(', ')
  }
}];
```
