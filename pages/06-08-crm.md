# 레슨 6.8: CRM 영업 자동화

---

## CRM 자동화가 필요한 이유

영업팀이 CRM에 데이터를 수동으로 입력하는 데 쏟는 시간: **하루 평균 2시간 이상**

이 시간을 자동화로 0으로 만들 수 있습니다.

---

## 지원 CRM 도구

n8n에서 네이티브 지원하는 주요 CRM:
- **HubSpot**: 무료 플랜 있음, 가장 인기
- **Salesforce**: 대기업 표준
- **Pipedrive**: 영업 중심 CRM
- **Zoho CRM**: 합리적 가격
- **Airtable**: 유연한 커스텀 CRM

## 프로젝트: 완전 자동화된 리드 관리

### 목표 워크플로우

```
[새 리드 발생 소스들]
  ├── 웹사이트 폼
  ├── LinkedIn 메시지
  ├── 이메일 문의
  └── 명함 촬영 (OCR)
        ↓
[리드 중복 확인]
        ↓
[AI 리드 스코어링]
        ↓
[CRM 등록 + 담당자 배정]
        ↓
[첫 접촉 이메일 자동 발송]
        ↓
[7일 후 팔로업 리마인더]
```

---

## STEP 1: 여러 소스에서 리드 수집

### 웹사이트 폼 (Webhook)

```
[Webhook: /lead/website]
    ↓
[Code: 데이터 정제]
  return [{
    json: {
      source: 'website',
      name: $json.name,
      email: $json.email,
      company: $json.company,
      phone: $json.phone,
      inquiry: $json.message
    }
  }];
```

### 이메일 리드 (Gmail Trigger)

```
[Gmail Trigger: 특정 라벨 "리드문의"]
    ↓
[AI: 이메일에서 연락처 정보 추출]
  Prompt: "이 이메일에서 이름, 이메일, 회사명, 전화번호를 JSON으로 추출해줘"
    ↓
[Code: 데이터 정제]
  source: 'email'
```

## STEP 2: 중복 리드 확인

```javascript
// HubSpot에서 이메일로 기존 연락처 검색
// [HubSpot: Search Contacts]
// Filter: email = {{ $json.email }}

// 결과 처리
const existingContact = $input.item.json;

if (existingContact.total > 0) {
  // 기존 연락처 업데이트
  return [{ json: { action: 'update', contactId: existingContact.results[0].id } }];
} else {
  // 새 연락처 생성
  return [{ json: { action: 'create' } }];
}
```

## STEP 3: AI 리드 스코어링

```javascript
// AI 프롬프트로 리드 품질 평가
const prompt = `
다음 리드 정보를 분석하고 1-100점으로 점수를 매겨줘.

이름: ${$json.name}
회사: ${$json.company}
문의 내용: ${$json.inquiry}
소스: ${$json.source}

점수 기준:
- 회사 규모 (큰 회사일수록 높은 점수)
- 문의 내용의 구체성 (구체적일수록 높음)
- 예산 언급 여부 (언급시 +20점)
- 긴급도 (즉시 필요 언급 시 +15점)

JSON 형식으로만 반환:
{
  "score": 75,
  "grade": "A",
  "reason": "예산이 충분하고...",
  "recommended_action": "즉시 전화 미팅 제안"
}
`;
```

## STEP 4: CRM 등록 및 담당자 자동 배정

```javascript
// 점수에 따른 담당자 배정
const score = $json.leadScore;
let assignee;

if (score >= 80) {
  assignee = { id: 'senior_sales_01', name: '김선임' };
} else if (score >= 60) {
  assignee = { id: 'sales_02', name: '이영업' };
} else {
  assignee = { id: 'junior_sales_03', name: '박주니어' };
}

return [{ json: { ...$json, assignee } }];
```

```
[HubSpot: Create Contact]
  Properties:
    firstname: {{ $json.name.split(' ')[0] }}
    lastname: {{ $json.name.split(' ')[1] || '' }}
    email: {{ $json.email }}
    company: {{ $json.company }}
    phone: {{ $json.phone }}
    lead_score: {{ $json.leadScore }}
    lead_source: {{ $json.source }}
    hubspot_owner_id: {{ $json.assignee.id }}
```

## STEP 5: 자동 첫 접촉 이메일

```
[Gmail: 첫 접촉 이메일 발송]
  To: {{ $json.email }}
  Subject: {{ $json.company }} - 문의 주셔서 감사합니다
  Body:
    안녕하세요 {{ $json.name }}님,

    {{ $json.company }}에서 저희 솔루션에 관심 가져주셔서 감사합니다.
    담당자 {{ $json.assignee.name }}가 영업일 기준 24시간 내 연락드릴 예정입니다.

    문의하신 내용:
    {{ $json.inquiry }}

    빠른 상담을 원하신다면 아래 링크에서 미팅을 예약하실 수 있습니다:
    [미팅 예약하기](https://calendly.com/your-company)

    감사합니다.
    영업팀 드림
```

---

## STEP 6: 팔로업 자동화

```
[Wait: 3 영업일]
    ↓
[HubSpot: 연락처 상태 확인]
  contactId: {{ $json.hubspotContactId }}
    ↓
[IF: 아직 연락 없음?]
  ├── True → [Slack: 담당자에게 팔로업 리마인더]
  │            "리마인더: {{ $json.name }}님 ({{ $json.company }}) 팔로업이 필요합니다"
  └── False → [처리 완료]
```

## 영업 파이프라인 자동 업데이트

```
[Webhook: 미팅 완료 이벤트 (Calendly)]
    ↓
[HubSpot: Deal 스테이지 업데이트]
  현재: "리드 접촉" → 다음: "미팅 완료"
    ↓
[HubSpot: 미팅 메모 추가]
[Slack: 영업팀에 미팅 완료 알림]
```

## 영업 성과 자동 리포트

```
[Schedule: 매주 금요일 오후 5시]
    ↓
[HubSpot: 이번 주 신규 리드 조회]
[HubSpot: 이번 주 성사된 Deal 조회]
    ↓
[Code: 주간 성과 계산]
    ↓
[Slack: #sales 채널에 주간 리포트]
```

## 핵심 요약

- 여러 소스(폼, 이메일, LinkedIn)의 리드를 하나의 워크플로우로 통합 처리
- AI 리드 스코어링으로 고품질 리드 우선 처리
- 점수 기반 자동 담당자 배정으로 적재적소 배치
- 첫 접촉부터 팔로업까지 전 과정 자동화
- HubSpot API로 파이프라인 스테이지 자동 업데이트
