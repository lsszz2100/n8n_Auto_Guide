## 핵심 개념 요약

### n8n의 강점
- 오픈소스 + 셀프 호스팅 = 무제한 무료 실행
- 코드 + 노코드 = 모든 사용자를 위한 플랫폼
- 400+ 통합 + AI 에이전트 = 강력한 자동화

### 설치 방법 선택 가이드

| 상황 | 방법 |
|------|------|
| 처음 시작 | n8n Cloud |
| 로컬 테스트 | npm 또는 Docker |
| 실운영 서버 | Docker Compose |
| 기업 | Docker + PostgreSQL + Nginx |

### 인터페이스 핵심

| 메뉴 | 역할 |
|------|------|
| Workflows | 워크플로우 목록 |
| Credentials | API 키/비밀번호 저장 |
| Executions | 실행 기록 |
| Settings | 시스템 설정 |

### 첫 워크플로우 구조
```
트리거 → 데이터 변환(Set) → 로직 처리(Code) → 결과 출력
```

## 초보자가 자주 하는 실수

### 실수 1: 테스트 URL을 프로덕션에서 사용한다

```
잘못된 방법:
  Webhook 테스트 URL을 외부 서비스에 등록

올바른 방법:
  테스트 URL → 개발 중 직접 테스트용
  프로덕션 URL → 워크플로우 활성화 후 외부 서비스에 등록
```

### 실수 2: 크리덴셜을 노드마다 새로 만든다

```
잘못된 방법:
  Gmail 노드마다 새 크리덴셜 생성

올바른 방법:
  크리덴셜 한 번 저장 → 모든 노드에서 같은 크리덴셜 재사용
  변경 시 한 곳만 수정하면 전체 적용
```

### 실수 3: 활성화 전에 충분히 테스트하지 않는다

```
체크리스트:
  [ ] 정상 데이터로 테스트 완료
  [ ] 빈 데이터, null 값으로 테스트
  [ ] 에러 상황 테스트
  [ ] 마지막 수동 실행 성공 확인
  → 이후에 활성화
```

## 설치 방법 빠른 참조

### n8n Cloud (추천: 처음 시작)
```
1. n8n.io → "Start for free"
2. 이메일 가입 → 자동 인스턴스 생성
3. 바로 사용 가능 (설치 없음)
```

### Docker (추천: 로컬/서버 운영)
```bash
# 단일 명령으로 실행
docker run -d \
  --name n8n \
  -p 5678:5678 \
  -v n8n_data:/home/node/.n8n \
  n8nio/n8n

# 접속: http://localhost:5678
```

### 중요 초기 설정 체크리스트
```
[ ] Settings → Timezone: Asia/Seoul 설정
[ ] Settings → Execution Data: 보관 기간 설정
[ ] 이메일로 가입하여 워크플로우 백업 활성화
[ ] 첫 워크플로우: Manual Trigger → Set 노드로 테스트
```
