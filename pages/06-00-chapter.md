## 6장을 시작하기 전 준비사항

### 필수 조건

- n8n이 설치되어 실행 중인 환경 (2장 완료)
- n8n 노드와 트리거의 기본 이해 (3장 완료)
- 워크플로우 설계 및 에러 처리 개념 (4장 완료)

### 추천 계정 준비

각 프로젝트에 필요한 계정을 미리 만들어두면 진행이 빠릅니다:

```
공통:
  - Google 계정 (Gmail, Sheets)
  
선택:
  - Slack 워크스페이스 (Slack 봇)
  - Telegram 계정 (Telegram 봇)
  - Notion 계정 (Notion 연동)
  - Twitter/X 개발자 계정 (소셜 미디어)
  - HubSpot 무료 계정 (CRM)
```

계정이 없는 서비스의 레슨은 건너뛰어도 됩니다. 각 프로젝트는 독립적입니다.

## 6장 학습 방법

### 단계별 따라하기

각 레슨을 진행할 때 이 순서를 지키세요:

```
1단계: 프로젝트 전체 구조 파악 (스크롤로 전체 훑어보기)
2단계: 필요한 크리덴셜 미리 발급
3단계: n8n에서 빈 워크플로우 생성
4단계: 설명을 보며 노드 하나씩 추가
5단계: 각 노드 추가 후 테스트 실행으로 데이터 흐름 확인
6단계: 전체 워크플로우 완성 후 최종 테스트
7단계: 워크플로우 활성화
```

### 막혔을 때

- 노드 설정 확인: 오른쪽 패널의 빨간 경고 메시지부터 읽기
- 데이터 흐름 확인: 이전 노드의 Output 탭에서 실제 데이터 구조 확인
- 표현식 오류: `{{ }}` 안의 경로가 맞는지 이전 노드 Output에서 직접 확인

## 각 프로젝트 미리보기

### 6.1 Gmail 자동화
수신 이메일을 자동 분류하고, 중요 이메일에 자동 답장 초안을 생성합니다.
실무에서 가장 많이 사용되는 자동화 중 하나입니다.

### 6.2 Slack 봇
명령어를 입력하면 업무 데이터를 조회하고 응답하는 Slack 봇을 만듭니다.
팀 내부에서 실제로 사용할 수 있는 수준으로 완성합니다.

### 6.3 Telegram 봇
개인 알림 봇부터 커맨드 기반 봇까지, Telegram API를 n8n과 연결합니다.

### 6.4 Google Sheets 자동화
스프레드시트를 데이터베이스처럼 활용하는 워크플로우를 구축합니다.
코딩 없이 CRUD(생성/조회/수정/삭제)를 자동화합니다.

### 6.5 Notion 연동
Notion 페이지 자동 생성, 데이터베이스 업데이트를 n8n으로 처리합니다.
프로젝트 관리 자동화의 핵심입니다.

### 6.6 소셜 미디어 자동 포스팅
콘텐츠를 한 번 작성하면 여러 플랫폼에 동시에 자동 게시합니다.

### 6.7 웹훅 기반 자동화
외부 서비스에서 이벤트가 발생할 때 즉시 반응하는 실시간 워크플로우를 만듭니다.

### 6.8 CRM 영업 자동화
리드 수집부터 이메일 발송, 팀 알림까지 영업 파이프라인을 완전 자동화합니다.
