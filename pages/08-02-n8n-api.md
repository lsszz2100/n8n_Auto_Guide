---

## n8n API란?

n8n은 REST API를 제공하여 외부 시스템에서 n8n을 프로그래밍 방식으로 제어할 수 있습니다.

**n8n API로 할 수 있는 것:**
- 워크플로우 목록 조회, 생성, 수정, 삭제
- 워크플로우 실행 및 중지
- 실행 이력 조회
- 크리덴셜 관리
- 태그 관리

---

## API 키 생성

```
n8n 화면 우측 상단 → 사용자 아이콘
→ Settings → API
→ "Create an API key"
→ 이름 지정 후 생성
→ 키 복사 (이후 재확인 불가!)
```

## 기본 API 사용법

### 베이스 URL

```
Self-hosted: https://your-n8n-domain.com/api/v1
n8n Cloud:   https://app.n8n.cloud/api/v1
```

### 인증 헤더

```
X-N8N-API-KEY: your-api-key-here
```

## 주요 API 엔드포인트

### 워크플로우 관리

```bash
# 워크플로우 목록 조회
GET /workflows

# 특정 워크플로우 조회
GET /workflows/{id}

# 워크플로우 생성
POST /workflows

# 워크플로우 수정
PUT /workflows/{id}

# 워크플로우 활성화/비활성화
POST /workflows/{id}/activate
POST /workflows/{id}/deactivate

# 워크플로우 삭제
DELETE /workflows/{id}
```

### 실행 관리

```bash
# 실행 목록 조회
GET /executions

# 특정 실행 조회
GET /executions/{id}

# 워크플로우 수동 실행
POST /workflows/{id}/run

# 실행 중지
POST /executions/{id}/stop

# 실행 삭제
DELETE /executions/{id}
```

## 실전 예시: 외부 앱에서 워크플로우 실행

### Python으로 n8n 워크플로우 호출

```python
import requests

N8N_URL = "https://your-n8n.com/api/v1"
API_KEY = "your-api-key"
WORKFLOW_ID = "abc123"

headers = {
    "X-N8N-API-KEY": API_KEY,
    "Content-Type": "application/json"
}

# 워크플로우 실행
response = requests.post(
    f"{N8N_URL}/workflows/{WORKFLOW_ID}/run",
    headers=headers,
    json={
        "startNodes": [],
        "destinationNode": "",
        "runData": {
            "Webhook": [
                {
                    "json": {
                        "customerId": "C001",
                        "action": "send_report"
                    }
                }
            ]
        }
    }
)

print(response.json())
# { "executionId": "xyz789" }
```

### JavaScript/Node.js로 실행

```javascript
const axios = require('axios');

const n8nApi = axios.create({
  baseURL: 'https://your-n8n.com/api/v1',
  headers: {
    'X-N8N-API-KEY': process.env.N8N_API_KEY
  }
});

// 워크플로우 실행
async function runWorkflow(workflowId, inputData) {
  const response = await n8nApi.post(
    `/workflows/${workflowId}/run`,
    {
      runData: {
        "Manual Trigger": [{ json: inputData }]
      }
    }
  );
  return response.data.executionId;
}

// 실행 결과 폴링
async function waitForCompletion(executionId, maxWait = 60000) {
  const start = Date.now();
  while (Date.now() - start < maxWait) {
    const exec = await n8nApi.get(`/executions/${executionId}`);
    if (exec.data.finished) {
      return exec.data;
    }
    await new Promise(r => setTimeout(r, 1000));
  }
  throw new Error('Timeout');
}
```

## 워크플로우 배포 자동화

### CI/CD 파이프라인에서 n8n 워크플로우 배포

```bash
#!/bin/bash
# deploy-workflow.sh

N8N_URL="https://your-n8n.com/api/v1"
API_KEY="$N8N_API_KEY"
WORKFLOW_FILE="workflows/email-automation.json"

# 워크플로우 JSON 읽기
WORKFLOW_JSON=$(cat "$WORKFLOW_FILE")

# 기존 워크플로우 ID 조회
EXISTING_ID=$(curl -s \
  -H "X-N8N-API-KEY: $API_KEY" \
  "$N8N_URL/workflows" | \
  jq -r ".data[] | select(.name == \"이메일 자동화\") | .id")

if [ -z "$EXISTING_ID" ]; then
  # 새로 생성
  curl -s -X POST \
    -H "X-N8N-API-KEY: $API_KEY" \
    -H "Content-Type: application/json" \
    -d "$WORKFLOW_JSON" \
    "$N8N_URL/workflows"
else
  # 기존 것 업데이트
  curl -s -X PUT \
    -H "X-N8N-API-KEY: $API_KEY" \
    -H "Content-Type: application/json" \
    -d "$WORKFLOW_JSON" \
    "$N8N_URL/workflows/$EXISTING_ID"
fi

echo "배포 완료!"
```

## 실행 이력 모니터링

```javascript
// 오늘 실패한 실행 조회
async function getFailedExecutions() {
  const today = new Date().toISOString().split('T')[0];

  const response = await n8nApi.get('/executions', {
    params: {
      status: 'error',
      startedAfter: `${today}T00:00:00.000Z`,
      limit: 50
    }
  });

  return response.data.data.map(exec => ({
    id: exec.id,
    workflowName: exec.workflowData?.name,
    startedAt: exec.startedAt,
    errorMessage: exec.data?.resultData?.error?.message
  }));
}

// 일일 실행 통계
async function getDailyStats() {
  const [allExec, errorExec] = await Promise.all([
    n8nApi.get('/executions', { params: { limit: 1 } }),
    n8nApi.get('/executions', { params: { status: 'error', limit: 1 } })
  ]);

  return {
    total: allExec.data.count,
    errors: errorExec.data.count,
    successRate: ((allExec.data.count - errorExec.data.count) /
                  allExec.data.count * 100).toFixed(1) + '%'
  };
}
```

## n8n API를 n8n 안에서 사용

워크플로우 안에서 다른 워크플로우를 동적으로 제어:

```
[Schedule: 매주 월요일 9시]
    ↓
[HTTP Request: 비활성 워크플로우 조회]
  Method: GET
  URL: {{ $vars.N8N_URL }}/api/v1/workflows
  Headers:
    X-N8N-API-KEY: {{ $vars.N8N_API_KEY }}
  Query Params:
    active: false
    ↓
[Code: 오래된 비활성 워크플로우 필터링]
  // 90일 이상 비활성화된 워크플로우 찾기
  const oldWorkflows = $input.all()
    .filter(item => {
      const updatedAt = new Date(item.json.updatedAt);
      const daysSince = (Date.now() - updatedAt) / 86400000;
      return daysSince > 90;
    });
    ↓
[Slack: 정리 필요한 워크플로우 목록 알림]
```

## API 보안 주의사항

- API 키는 환경 변수나 n8n Variables로 관리 (하드코딩 금지)
- API 키에 최소 권한 원칙 적용
- HTTPS만 사용
- Rate limiting 고려 (n8n API도 속도 제한 있음)
- API 키 정기적 교체
