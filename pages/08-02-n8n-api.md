# 레슨 8.2: n8n API 활용하기

---

## n8n API란?

n8n은 REST API를 제공하여 외부 시스템에서 n8n을 프로그래밍 방식으로 제어할 수 있습니다.

**n8n API로 할 수 있는 것:**
- 워크플로우 목록 조회, 생성, 수정, 삭제
- 워크플로우 실행 및 중지
- 실행 이력 조회
- 크리덴셜 관리
- 태그 관리

---

## API 키 생성

```
n8n 화면 우측 상단 → 사용자 아이콘
→ Settings → API
→ "Create an API key"
→ 이름 지정 후 생성
→ 키 복사 (이후 재확인 불가!)
```

## 주요 API 엔드포인트

### 워크플로우 관리

```bash
# 워크플로우 목록 조회
GET /workflows

# 특정 워크플로우 조회
GET /workflows/{id}

# 워크플로우 생성
POST /workflows

# 워크플로우 수정
PUT /workflows/{id}

# 워크플로우 활성화/비활성화
POST /workflows/{id}/activate
POST /workflows/{id}/deactivate

# 워크플로우 삭제
DELETE /workflows/{id}
```

### 실행 관리

```bash
# 실행 목록 조회
GET /executions

# 특정 실행 조회
GET /executions/{id}

# 워크플로우 수동 실행
POST /workflows/{id}/run

# 실행 중지
POST /executions/{id}/stop

# 실행 삭제
DELETE /executions/{id}
```

## 실전 예시: 외부 앱에서 워크플로우 실행

### Python으로 n8n 워크플로우 호출

```python
import requests

N8N_URL = "https://your-n8n.com/api/v1"
API_KEY = "your-api-key"
WORKFLOW_ID = "abc123"

headers = {
    "X-N8N-API-KEY": API_KEY,
    "Content-Type": "application/json"
}

# 워크플로우 실행
response = requests.post(
    f"{N8N_URL}/workflows/{WORKFLOW_ID}/run",
    headers=headers,
    json={
        "startNodes": [],
        "destinationNode": "",
        "runData": {
            "Webhook": [
                {
                    "json": {
                        "customerId": "C001",
                        "action": "send_report"
                    }
                }
            ]
        }
    }
)

print(response.json())
# { "executionId": "xyz789" }
```

### JavaScript/Node.js로 실행

```javascript
const axios = require('axios');

const n8nApi = axios.create({
  baseURL: 'https://your-n8n.com/api/v1',
  headers: {
    'X-N8N-API-KEY': process.env.N8N_API_KEY
  }
});

// 워크플로우 실행
async function runWorkflow(workflowId, inputData) {
  const response = await n8nApi.post(
    `/workflows/${workflowId}/run`,
    {
      runData: {
        "Manual Trigger": [{ json: inputData }]
      }
    }
  );
  return response.data.executionId;
}

// 실행 결과 폴링
async function waitForCompletion(executionId, maxWait = 60000) {
  const start = Date.now();
  while (Date.now() - start < maxWait) {
    const exec = await n8nApi.get(`/executions/${executionId}`);
    if (exec.data.finished) {
      return exec.data;
    }
    await new Promise(r => setTimeout(r, 1000));
  }
  throw new Error('Timeout');
}
```

## 워크플로우 배포 자동화

### CI/CD 파이프라인에서 n8n 워크플로우 배포

```bash
#!/bin/bash
# deploy-workflow.sh

N8N_URL="https://your-n8n.com/api/v1"
API_KEY="$N8N_API_KEY"
WORKFLOW_FILE="workflows/email-automation.json"

# 워크플로우 JSON 읽기
WORKFLOW_JSON=$(cat "$WORKFLOW_FILE")

# 기존 워크플로우 ID 조회
EXISTING_ID=$(curl -s \
  -H "X-N8N-API-KEY: $API_KEY" \
  "$N8N_URL/workflows" | \
  jq -r ".data[] | select(.name == \"이메일 자동화\") | .id")

if [ -z "$EXISTING_ID" ]; then
  # 새로 생성
  curl -s -X POST \
    -H "X-N8N-API-KEY: $API_KEY" \
    -H "Content-Type: application/json" \
    -d "$WORKFLOW_JSON" \
    "$N8N_URL/workflows"
else
  # 기존 것 업데이트
  curl -s -X PUT \
    -H "X-N8N-API-KEY: $API_KEY" \
    -H "Content-Type: application/json" \
    -d "$WORKFLOW_JSON" \
    "$N8N_URL/workflows/$EXISTING_ID"
fi

echo "배포 완료!"
```

## 실행 이력 모니터링

```javascript
// 오늘 실패한 실행 조회
async function getFailedExecutions() {
  const today = new Date().toISOString().split('T')[0];

  const response = await n8nApi.get('/executions', {
    params: {
      status: 'error',
      startedAfter: `${today}T00:00:00.000Z`,
      limit: 50
    }
  });

  return response.data.data.map(exec => ({
    id: exec.id,
    workflowName: exec.workflowData?.name,
    startedAt: exec.startedAt,
    errorMessage: exec.data?.resultData?.error?.message
  }));
}

// 일일 실행 통계
async function getDailyStats() {
  const [allExec, errorExec] = await Promise.all([
    n8nApi.get('/executions', { params: { limit: 1 } }),
    n8nApi.get('/executions', { params: { status: 'error', limit: 1 } })
  ]);

  return {
    total: allExec.data.count,
    errors: errorExec.data.count,
    successRate: ((allExec.data.count - errorExec.data.count) /
                  allExec.data.count * 100).toFixed(1) + '%'
  };
}
```

```
  // 90일 이상 비활성화된 워크플로우 찾기
  const oldWorkflows = $input.all()
    .filter(item => {
      const updatedAt = new Date(item.json.updatedAt);
      const daysSince = (Date.now() - updatedAt) / 86400000;
      return daysSince > 90;

```

### 예제 1: 워크플로우 실행 이력 조회 + 실패 건 Slack 알림

매 시간마다 n8n API로 최근 실행 이력을 조회하고, 실패한 건이 있으면 담당자에게 Slack 알림을 보내는 모니터링 워크플로우입니다.

```
워크플로우 이름: "실행 실패 감지 및 알림"
트리거: Schedule (매 1시간)

[Schedule Trigger]
  Rule: 매 시간 정각
    ↓
[HTTP Request: 최근 1시간 실행 이력 조회]
  Method: GET
  URL: {{ $vars.N8N_BASE_URL }}/api/v1/executions
  Headers:
    X-N8N-API-KEY: {{ $vars.N8N_API_KEY }}
  Query Parameters:
    status: error
    limit: 100
    startedAfter: {{ $now.minus(1, 'hour').toISO() }}
    ↓
[Code: 실패 건 정리]
  const executions = $json.data || [];

  if (executions.length === 0) {
    return [{ json: { hasFailures: false, count: 0 } }];
  }

  // 워크플로우별로 그룹화
  const grouped = {};
  for (const exec of executions) {
    const name = exec.workflowData?.name || "알 수 없음";
    if (!grouped[name]) {
      grouped[name] = { name, count: 0, execIds: [], lastError: "" };
    }
    grouped[name].count++;
    grouped[name].execIds.push(exec.id);
    grouped[name].lastError = exec.data?.resultData?.error?.message || "오류 정보 없음";
  }

  const summaryLines = Object.values(grouped).map(g =>
    `- ${g.name}: ${g.count}건 실패 (마지막 오류: ${g.lastError.slice(0, 60)})`
  );

  return [{
    json: {
      hasFailures: true,
      count: executions.length,
      summary: summaryLines.join("\n"),
      period: "최근 1시간"
    }
  }];
    ↓
[IF: hasFailures === true]
  False → [종료: 실패 없음]
  True →
    [Slack: 실패 알림 발송]
      Channel: #n8n-failures
      Message: |
        워크플로우 실행 실패 감지 ({{ $json.period }})
        총 {{ $json.count }}건 실패

        워크플로우별 상세:
        {{ $json.summary }}

        n8n 대시보드에서 확인:
        {{ $vars.N8N_BASE_URL }}/executions
```

### 예제 2: 외부 대시보드에서 n8n 워크플로우 원격 트리거

사내 관리 대시보드(Next.js)에서 버튼 클릭으로 n8n 워크플로우를 원격 실행하고, 실행 결과를 폴링으로 확인하는 패턴입니다.

n8n 워크플로우 구성 (Webhook 방식 원격 트리거용):

```
워크플로우 이름: "원격 보고서 생성"
트리거: Webhook

[Webhook Trigger]
  Method: POST
  Path: /generate-report
  Authentication: Header Auth
    Header: X-Trigger-Secret
    Value: {{ $vars.WEBHOOK_SECRET }}
    ↓
[Code: 요청 파라미터 검증]
  const { reportType, startDate, endDate, requesterId } = $json.body;

  if (!reportType || !startDate || !endDate) {
    return [{
      json: {
        error: true,
        message: "reportType, startDate, endDate 필수",
        status: 400
      }
    }];
  }

  return [{
    json: { reportType, startDate, endDate, requesterId }
  }];
    ↓
[Switch: reportType]
  "sales"     → [Google Sheets: 매출 데이터 조회]
  "inventory" → [DB: 재고 현황 조회]
  "customer"  → [DB: 고객 분석 쿼리]
    ↓
[Code: 보고서 CSV 생성]
  // 조회 데이터를 CSV 형식으로 변환
  const data = $input.all();
  const headers = Object.keys(data[0].json).join(",");
  const rows = data.map(item => Object.values(item.json).join(","));
  const csv = [headers, ...rows].join("\n");

  return [{ json: { csv, rowCount: data.length } }];
    ↓
[Google Drive: 보고서 파일 업로드]
  File Name: {{ $json.reportType }}-report-{{ $now.format('YYYYMMDD') }}.csv
  Content: {{ $json.csv }}
  Folder: "자동화 보고서"
    ↓
[Webhook Response: 결과 반환]
  Status: 200
  Body:
    {
      "success": true,
      "reportUrl": "{{ $json.webViewLink }}",
      "rowCount": {{ $json.rowCount }},
      "generatedAt": "{{ $now.toISO() }}"
    }
```

외부 Next.js 대시보드에서 호출하는 코드:

```javascript
// pages/api/trigger-report.js
export default async function handler(req, res) {
  const { reportType, startDate, endDate } = req.body;

  const response = await fetch(
    `${process.env.N8N_BASE_URL}/webhook/generate-report`,
    {
      method: "POST",
      headers: {
        "Content-Type": "application/json",
        "X-Trigger-Secret": process.env.WEBHOOK_SECRET
      },
      body: JSON.stringify({
        reportType,
        startDate,
        endDate,
        requesterId: req.session.userId
      })
    }
  );

  const result = await response.json();

  if (result.success) {
    res.json({
      message: "보고서 생성 완료",
      downloadUrl: result.reportUrl,
      rowCount: result.rowCount
    });
  } else {
    res.status(500).json({ error: "보고서 생성 실패" });
  }
}
```

### 예제 3: 워크플로우 목록 조회 + 비활성 워크플로우 자동 정리

주기적으로 비활성화된 워크플로우를 조회하고, 90일 이상 미수정된 워크플로우를 자동으로 아카이브(태그 추가 + 담당자 알림)하는 워크플로우입니다.

```
워크플로우 이름: "비활성 워크플로우 정리"
트리거: Schedule (매주 월요일 오전 10시)

[Schedule Trigger]
  Rule: 0 10 * * 1
    ↓
[HTTP Request: 전체 워크플로우 목록 조회]
  Method: GET
  URL: {{ $vars.N8N_BASE_URL }}/api/v1/workflows
  Headers:
    X-N8N-API-KEY: {{ $vars.N8N_API_KEY }}
  Query Parameters:
    active: false
    limit: 250
    ↓
[Code: 오래된 비활성 워크플로우 필터링]
  const workflows = $json.data || [];
  const NOW = Date.now();
  const THRESHOLD_DAYS = 90;
  const THRESHOLD_MS = THRESHOLD_DAYS * 24 * 60 * 60 * 1000;

  const oldInactive = workflows.filter(wf => {
    const updatedAt = new Date(wf.updatedAt).getTime();
    const age = NOW - updatedAt;
    return age > THRESHOLD_MS;
  }).map(wf => ({
    id: wf.id,
    name: wf.name,
    updatedAt: wf.updatedAt,
    daysSinceUpdate: Math.floor((NOW - new Date(wf.updatedAt).getTime()) / 86400000),
    tags: wf.tags?.map(t => t.name) || []
  }));

  return oldInactive.length > 0
    ? oldInactive.map(wf => ({ json: wf }))
    : [{ json: { noTargets: true } }];
    ↓
[IF: noTargets === true]
  True → [Slack: 정리 대상 없음 알림] → [종료]
  False →
    [Split In Batches: 1개씩]
      ↓
    [HTTP Request: 태그 추가 (아카이브 표시)]
      Method: PATCH
      URL: {{ $vars.N8N_BASE_URL }}/api/v1/workflows/{{ $json.id }}
      Headers:
        X-N8N-API-KEY: {{ $vars.N8N_API_KEY }}
      Body:
        {
          "tags": [{ "name": "archived-auto" }]
        }
      ↓
    [다음 워크플로우 처리...]
    ↓
[Merge: 처리 완료 후 합산]
    ↓
[Code: 정리 결과 요약]
  const processed = $input.all();
  const names = processed.map(i => `- ${i.json.name} (${i.json.daysSinceUpdate}일 미수정)`);
  return [{
    json: {
      count: processed.length,
      list: names.join("\n")
    }
  }];
    ↓
[Slack: 정리 완료 보고]
  Channel: #ops-weekly
  Message: |
    비활성 워크플로우 자동 정리 완료

    90일 이상 미수정 워크플로우 {{ $json.count }}개에
    "archived-auto" 태그를 추가했습니다.

    대상 목록:
    {{ $json.list }}

    필요하다면 n8n에서 태그를 제거해 복원하세요.
    {{ $vars.N8N_BASE_URL }}/workflows
```
