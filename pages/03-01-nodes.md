# 레슨 3.1: 노드(Node) 완전 이해

---

## 노드란?

**노드(Node)**는 n8n 워크플로우의 기본 단위입니다.

각 노드는 **하나의 작업**을 수행합니다:
- 데이터 가져오기
- 데이터 변환하기
- 메시지 보내기
- 조건 판단하기
- 등등

워크플로우는 이 노드들을 **연결한 체인**입니다.

```
[노드 A] → [노드 B] → [노드 C]
  데이터   →  변환  →   전송
```

---

## 노드의 두 가지 유형

### 1. 트리거 노드 (Trigger Nodes)

워크플로우를 **시작**하는 노드입니다. 모든 워크플로우는 반드시 하나의 트리거 노드로 시작해야 합니다.

**특징**:
- 주황색 번개 아이콘 ⚡로 표시
- 워크플로우당 하나만 가능
- 항상 첫 번째 노드

**종류**:
- Manual Trigger (수동 실행)
- Webhook (HTTP 요청 수신)
- Schedule Trigger (시간 기반)
- Email Trigger (이메일 수신)
- Gmail Trigger, Slack Trigger 등 앱별 트리거

### 2. 일반 노드 (Regular Nodes)

트리거 이후에 실행되는 **처리 노드**입니다.

**특징**:
- 이전 노드에서 데이터를 받아 처리
- 여러 개를 연결 가능
- 분기, 병합, 변환, 전송 등 다양한 역할

---

## 노드의 내부 구조

모든 노드는 세 가지 핵심 부분으로 구성됩니다:

### 1. Parameters (파라미터)
노드의 **설정값**입니다. 노드가 무엇을 할지를 정의합니다.

```
Gmail 노드의 파라미터 예시:
- To: 받는 사람 이메일
- Subject: 제목
- Message: 내용
- CC: 참조
```

### 2. Input / Output (입출력)
- **Input**: 이전 노드에서 받은 데이터
- **Output**: 처리 후 다음 노드로 보낼 데이터

```
이전 노드 아웃풋  →  현재 노드 인풋  →  처리  →  현재 노드 아웃풋  →  다음 노드
```

### 3. Credentials (크리덴셜)
외부 서비스에 연결하기 위한 **인증 정보**입니다. (API 키, OAuth 토큰 등)
크리덴셜은 한 번 저장하면 여러 노드에서 재사용 가능합니다.

---

## 노드 연결 이해하기

노드와 노드는 **연결선(Connection)**으로 이어집니다.

```
[노드 A]──────────────►[노드 B]
```

### 연결의 의미
연결선은 데이터의 흐름을 나타냅니다.
노드 A의 출력 데이터가 그대로 노드 B의 입력 데이터가 됩니다.

### 다중 연결
한 노드에서 여러 노드로 연결 가능합니다 (분기):
```
[트리거]─────►[이메일 발송]
         └───►[Slack 알림]
```

여러 노드에서 한 노드로 연결도 가능합니다 (병합):
```
[소스 A]─────►[Merge]───►[처리]
[소스 B]─────►
```

## 아이템(Item) 개념

n8n에서 데이터는 **아이템(Item)**의 배열 형태로 흐릅니다.

각 아이템은 하나의 JSON 객체입니다.

```
아이템 배열의 예시:
[
  { "name": "홍길동", "email": "hong@test.com" },  ← 아이템 1
  { "name": "김철수", "email": "kim@test.com" },   ← 아이템 2
  { "name": "이영희", "email": "lee@test.com" }    ← 아이템 3
]
```

**중요**: 대부분의 노드는 아이템을 **하나씩** 처리합니다. 3개의 아이템이 들어오면 3번 실행됩니다.

---

## 노드 실행 상태

워크플로우 실행 후 각 노드의 상태를 시각적으로 확인할 수 있습니다:

| 상태 | 아이콘 | 의미 |
|------|--------|------|
| 성공 | ✅ 초록색 | 정상 실행 완료 |
| 실패 | ❌ 빨간색 | 에러 발생 |
| 대기 | ⏸️ 회색 | 아직 실행 안 됨 |
| 실행 중 | 🔄 파란색 | 현재 처리 중 |

노드를 클릭하면 처리된 아이템 수와 각 아이템의 데이터를 확인할 수 있습니다.

---

## 노드 설정 시 자주 사용하는 기능

### Expression 모드
파라미터 값을 고정값 대신 **동적 표현식**으로 설정합니다.

필드 오른쪽의 `fx` 아이콘 클릭으로 전환:
- 고정값 모드: `홍길동`
- 표현식 모드: `{{ $json.name }}`

### Add Always Fields
노드 실행 후 기존 데이터를 유지할지 덮어쓸지 선택합니다.
- **Replace**: 새 데이터로 완전 교체
- **Merge**: 기존 데이터에 새 필드 추가

---

## 노드 조작 팁

### 노드 비활성화
노드 우클릭 → "Disable": 해당 노드를 건너뛰고 실행
테스트 시 특정 단계를 임시로 건너뛸 때 유용

### 노드 이름 변경
노드 더블클릭 또는 설정 패널 상단 이름 클릭
의미 있는 이름을 붙이면 워크플로우를 이해하기 쉬워짐

### 노드 메모 추가
Settings 탭 → Notes: 노드에 설명을 추가
팀원이나 미래의 나를 위해 작성하는 습관을 들이세요

### 노드에 색상 지정
우클릭 → Change color: 노드 그룹을 색으로 구분하면 가독성 향상

---

## 핵심 요약

- 노드는 워크플로우의 기본 단위로, 하나의 작업을 수행
- 트리거 노드는 시작점, 일반 노드는 처리 단계
- 모든 노드는 파라미터, 입출력, 크리덴셜의 세 부분으로 구성
- 데이터는 아이템 배열 형태로 흐르며, 대부분의 노드는 아이템을 하나씩 처리
- 표현식 모드로 동적 데이터를 파라미터에 활용 가능
