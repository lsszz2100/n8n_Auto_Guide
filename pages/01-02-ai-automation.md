# 레슨 1.2: AI와 자동화

---

## 모든 자동화가 AI는 아니다

흔히 "자동화 = AI"라고 생각하지만, 이는 정확하지 않습니다.

전통적 자동화(Rule-based Automation)와 AI 자동화는 근본적으로 다릅니다.

---

## 규칙 기반 자동화 vs AI 자동화

### 규칙 기반 자동화 (Rule-based)

```
IF 주문 금액 > 50,000원 THEN 무료 배송
IF 이메일 제목에 "환불" 포함 THEN CS팀 전달
IF 재고 < 10개 THEN 발주 알림
```

특징:
- 명확한 조건과 결과
- 예외 없이 정확히 규칙을 따름
- 구축이 빠르고 비용이 낮음
- 예상치 못한 상황에 대처 불가

적합한 상황: 조건이 명확하고 고정된 프로세스

---

### AI 자동화 (AI-powered Automation)

```
이메일 → AI가 내용 분석 → 의도 파악 → 적절한 담당자 배정 + 초안 작성
고객 리뷰 → AI가 감정 분석 → 긍정/부정/중립 분류 → 대응 방법 결정
```

특징:
- 문맥을 이해하고 의미를 파악
- 애매한 상황에서도 판단 가능
- 경험을 통해 개선 가능
- 구축이 복잡하고 비용이 높음

적합한 상황: 판단과 해석이 필요한 비정형 데이터 처리

---

## 두 방식을 함께 사용하는 것이 최선

실제 비즈니스에서는 두 가지를 조합해서 사용할 때 가장 강력합니다.

```
[규칙] 이메일 수신
    ↓
[AI] 내용 분석 및 분류
    ↓
[규칙] 분류 결과에 따라 적절한 담당자에게 전달
    ↓
[AI] 초안 답변 생성
    ↓
[규칙] 담당자에게 검토 요청 발송
```

n8n은 이 두 가지 방식을 하나의 워크플로우에서 자연스럽게 결합할 수 있습니다.

---

## AI 자동화의 4가지 핵심 기능

### 1. 텍스트 이해 (NLP)
- 이메일, 메시지, 문서의 의미를 이해
- 감정, 의도, 주제 파악
- 언어 번역 자동화

### 2. 이미지 인식 (Vision)
- 영수증, 명함, 문서 데이터 자동 추출
- 제품 이미지 분류
- 품질 검사 자동화

### 3. 생성 (Generation)
- 이메일 초안 자동 작성
- 보고서 자동 생성
- 소셜 미디어 콘텐츠 생성

### 4. 추론 및 의사결정 (Reasoning)
- 복잡한 조건을 판단
- 여러 요소를 고려한 최적 선택
- 다음 행동 계획 수립

## n8n에서의 AI 자동화

n8n은 2024년부터 AI 기능을 대폭 강화하여, 현재 다음과 같은 AI 통합을 제공합니다:

| AI 서비스 | n8n 통합 방식 |
|-----------|--------------|
| OpenAI (GPT-4o, o3) | 네이티브 노드 |
| Anthropic (Claude 4 시리즈) | 네이티브 노드 |
| Google (Gemini) | 네이티브 노드 |
| 로컬 LLM (Ollama) | HTTP 연동 |
| Hugging Face | 네이티브 노드 |

## 실제 비즈니스 사례

### 사례 1: 전자상거래 고객 지원
Before: CS 팀원이 하루 200개 문의를 수동으로 읽고 분류
After: n8n + Claude가 자동으로 분류, 단순 문의는 AI가 즉시 답변, 복잡한 문의만 사람에게 전달
결과: CS 팀의 업무량 70% 감소, 평균 응답 시간 4시간 → 3분

### 사례 2: 콘텐츠 마케팅 자동화
Before: 마케터가 매주 5개 SNS에 각각 다른 형식으로 콘텐츠 작성
After: 하나의 아이디어를 입력하면 n8n + GPT-4가 각 플랫폼에 최적화된 형태로 변환하여 예약 발행
결과: 주간 콘텐츠 작성 시간 8시간 → 30분

### 사례 3: 영업 리드 관리
Before: 영업팀이 새 리드를 수동으로 검토하고 등급 부여
After: n8n + AI가 리드 정보 수집, 점수화, CRM 입력, 담당 영업사원 배정까지 자동화
결과: 리드 처리 속도 10배 향상

## AI 자동화 도입 시 고려사항

### 언제 AI를 써야 하는가?
- 텍스트나 이미지 분석이 필요할 때
- 판단과 해석이 필요할 때
- 규칙으로 표현하기 너무 복잡할 때

### 언제 규칙 기반을 써야 하는가?
- 조건이 단순하고 명확할 때
- 100% 정확도가 필요할 때
- 비용과 속도가 중요할 때

### AI 자동화의 한계
- AI는 틀릴 수 있습니다 (hallucination)
- 중요한 결정은 여전히 사람의 검토가 필요합니다
- 개인정보를 AI에 보낼 때는 데이터 보안을 반드시 고려해야 합니다

---

## 핵심 요약

- 자동화 ≠ AI. 규칙 기반 자동화와 AI 자동화는 다른 개념
- 둘을 조합하면 가장 강력한 자동화 시스템을 만들 수 있음
- n8n은 OpenAI, Claude, Gemini 등 모든 주요 AI와 연동 가능
- AI 자동화는 텍스트 이해, 이미지 인식, 생성, 추론의 4가지 기능이 핵심
