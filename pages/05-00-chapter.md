## 왜 AI 에이전트인가?

### 기존 자동화의 한계

지금까지 배운 자동화는 강력하지만, 한계가 있습니다.

| 상황 | 기존 자동화 | AI 에이전트 |
|------|------------|------------|
| "배송이 언제 되나요?" | 미리 정의된 답변만 가능 | 주문 정보를 조회해 맞춤 답변 |
| 이메일 분류 | 키워드로만 분류 | 내용을 이해해 분류 |
| 보고서 작성 | 템플릿 채우기 | 데이터를 분석해 통찰 생성 |
| 코드 에러 처리 | 에러 코드별 처리 | 에러 내용을 이해하고 해결책 제시 |

### n8n + AI = 최강 조합

n8n은 단순한 AI 호출 도구가 아닙니다. AI의 판단과 기존 시스템의 연동을 하나의 워크플로우로 통합합니다.

```
[이메일 수신] → [AI가 내용 분석] → [적절한 부서로 자동 라우팅]
                                  → [자동 답변 생성]
                                  → [CRM에 이슈 등록]
```

이 모든 것이 코드 한 줄 없이 가능합니다.
