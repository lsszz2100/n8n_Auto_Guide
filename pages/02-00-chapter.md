## 어떤 설치 방법을 선택해야 할까?

2장에는 세 가지 설치 방법이 나옵니다. 먼저 나에게 맞는 방법을 선택하세요:

```
질문 1: 빠르게 시작하고, 서버 관리를 원하지 않는다
  → n8n Cloud (레슨 2.3)
  
질문 2: 데이터를 내 서버에 보관하고 싶고, Docker를 알고 있다
  → Docker 셀프 호스팅 (레슨 2.4)
  
질문 3: 일단 로컬 PC에서 가볍게 테스트해보고 싶다
  → npm 로컬 설치 (레슨 2.5)
```

### 선택 가이드

| 상황 | 추천 방법 |
|------|---------|
| 처음 n8n을 써보는 경우 | n8n Cloud |
| 회사 업무에 바로 도입하려는 경우 | n8n Cloud 또는 Docker |
| 개인 서버(VPS)에 올리려는 경우 | Docker |
| 로컬에서 개발/테스트 목적 | npm |
| 비용을 아끼고 싶은 경우 | Docker (서버 비용만 발생) |

---

## n8n Cloud vs Self-Hosting 핵심 비교

| 항목 | n8n Cloud | 셀프 호스팅 |
|------|-----------|-----------|
| 설치 시간 | 5분 | 30분~1시간 |
| 서버 관리 | 불필요 | 직접 관리 |
| 비용 | 월 $20~ | 서버 비용만 |
| 데이터 위치 | n8n 서버 | 내 서버 |
| 업데이트 | 자동 | 수동 |
| 커스터마이징 | 제한적 | 자유로움 |
| 추천 대상 | 빠른 시작 | 장기 운영 |
