## 핵심 개념 요약

### 자동화란?
소프트웨어가 반복적인 작업을 사람 대신 실행하도록 만드는 것.
"X가 발생하면, Y를 실행해서, Z 결과를 얻는다"로 표현 가능한 모든 것이 자동화 후보.

### AI 자동화 vs 규칙 기반 자동화
| | 규칙 기반 | AI 자동화 |
|--|----------|-----------|
| 작동 방식 | 명확한 IF/THEN 규칙 | 문맥 이해 및 판단 |
| 비용 | 낮음 | 높음 |
| 유연성 | 낮음 | 높음 |
| 정확도 | 100% (규칙 내에서) | 높지만 완벽하지 않음 |
| 최적 사용 | 단순하고 명확한 프로세스 | 복잡하고 비정형적인 데이터 |

### 워크플로우의 3요소
1. 트리거: 워크플로우를 시작하는 이벤트
2. 액션: 실행되는 작업
3. 조건: 흐름을 제어하는 분기 로직

### API란?
앱들 사이의 중개인. 두 서비스가 서로 통신할 수 있게 해주는 규칙.
- GET (읽기) / POST (생성) / PUT (수정) / DELETE (삭제)
- Webhook: 외부에서 우리에게 먼저 알려주는 방식

### 자동화 후보 우선순위화
RICE 프레임워크 = (Reach × Impact × Confidence) / Effort

## 초보자가 자주 하는 실수

### 실수 1: 자동화 대상을 잘못 선택한다

```
잘못된 선택:
  "회의에서 무엇을 결정할지 자동화"
  → 창의적 판단이 필요한 업무는 자동화 불가

올바른 선택:
  "회의 전 참석자에게 안건 이메일 발송"
  → 반복적이고 규칙이 명확한 작업
```

### 실수 2: 자동화를 너무 크게 설계한다

```
잘못된 접근:
  처음부터 "전체 마케팅 프로세스 자동화" 시도

올바른 접근:
  1단계: 이메일 알림 자동화 (작게 시작)
  2단계: 데이터 수집 자동화 추가
  3단계: 리포트 생성 자동화 추가
  → 단계적으로 확장
```

### 실수 3: ROI를 계산하지 않는다

```
자동화 전에 반드시 계산:
  절약 시간 = (현재 소요 시간 × 빈도) - 구축/유지 시간
  
  예시:
  - 현재: 주문 확인 이메일 발송 30분 × 매일 = 월 15시간
  - 자동화 구축: 2시간
  - 월간 유지: 0.5시간
  → 첫 달부터 12.5시간 절약
```
