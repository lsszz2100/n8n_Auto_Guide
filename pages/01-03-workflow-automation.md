# 레슨 1.3: 워크플로우 자동화

---

## 워크플로우란?

워크플로우(Workflow)는 특정 목표를 달성하기 위한 일련의 작업 단계들의 연결입니다.

수동으로 수행하던 업무 흐름을 소프트웨어가 자동으로 실행하도록 만드는 것이 워크플로우 자동화입니다.

---

## 워크플로우의 3대 구성요소

모든 워크플로우는 세 가지 핵심 요소로 이루어집니다.

### 1. 트리거 (Trigger) — "무엇이 시작을 촉발하는가?"

트리거는 워크플로우를 시작시키는 이벤트입니다.

```
이메일 수신
구글 폼 제출
매일 오전 9시
새 Slack 메시지
결제 완료
재고 부족
```

### 2. 액션 (Action) — "무엇을 실행하는가?"

트리거가 발동되면 실행되는 실제 작업들입니다.

```
Google Sheets에 데이터 기록
이메일 발송
Slack 메시지 전송
Notion 페이지 생성
데이터 변환 및 처리
전화 알림 발송
```

### 3. 조건 (Condition) — "어떤 상황에서 어떻게 행동하는가?"

워크플로우의 흐름을 제어하는 분기점입니다.

```
IF 주문금액 > 100,000원 → VIP 처리 경로
ELSE → 일반 처리 경로

IF 이메일이 스팸 → 삭제
IF 이메일이 문의 → CS팀 전달
IF 이메일이 주문 → 영업팀 전달
```

---

## 워크플로우 자동화 시각화

실제 비즈니스 워크플로우를 시각적으로 이해해 봅시다.

### 예시: 신규 고객 온보딩 자동화

```
[트리거] 회원가입 완료
    ↓
[액션] CRM에 고객 정보 저장
    ↓
[액션] 웰컴 이메일 발송
    ↓
[조건] 요금제는?
    ├── 무료 → [액션] 무료 가이드 발송
    └── 유료 → [액션] 담당 CS 배정 + 온보딩 미팅 예약
```

### 예시: 주문 처리 자동화

```
[트리거] 새 주문 접수
    ↓
[액션] 재고 시스템 조회
    ↓
[조건] 재고 있음?
    ├── 예 → [액션] 배송 처리 + 고객 알림
    └── 아니오 → [액션] 고객에게 품절 안내 + 입고 알림 등록
```

## n8n에서의 워크플로우

n8n은 이 워크플로우를 시각적인 캔버스 위에서 구축합니다.

각 단계는 노드(Node)로 표현되며, 노드를 연결하는 선(Connection)이 데이터의 흐름을 나타냅니다.

```
[Webhook 트리거] → [IF 조건] → [Gmail 발송]
                         ↓
                   [Slack 알림]
```

이렇게 만들어진 워크플로우는:
- 저장해두면 자동으로 반복 실행됩니다
- 언제든 수정하고 업데이트할 수 있습니다
- 실행 기록을 모두 확인할 수 있습니다

---

## 워크플로우 자동화가 주는 핵심 가치

### 1. 시간 절약
반복 작업에서 해방되어 더 가치 있는 일에 집중할 수 있습니다.

### 2. 오류 감소
사람이 직접 하면 실수가 생기지만, 자동화는 매번 정확하게 실행됩니다.

### 3. 확장성 (Scalability)
사람을 더 고용하지 않아도 처리량을 쉽게 늘릴 수 있습니다.

### 4. 24/7 운영
자동화는 쉬지 않습니다. 주말, 새벽, 공휴일에도 동일하게 작동합니다.

### 5. 데이터 일관성
모든 데이터가 같은 방식으로 처리되어 일관된 품질을 유지합니다.

## 좋은 워크플로우의 조건

워크플로우를 만들기 전에 다음 세 가지를 명확히 해야 합니다:

### 1. 입력(Input)이 무엇인가?
- 어떤 데이터가 들어오는가?
- 데이터의 형식은?
- 언제 들어오는가?

### 2. 처리(Process) 단계는?
- 어떤 순서로 처리하는가?
- 어떤 조건 분기가 있는가?
- 어떤 변환이 필요한가?

### 3. 출력(Output)이 무엇인가?
- 최종적으로 무엇이 생성되어야 하는가?
- 어디로 전달되어야 하는가?
- 성공의 기준은?

> 황금 법칙: "X가 발생했을 때, Y를 해서, Z를 얻는다"를 명확히 정의할 수 있어야 좋은 워크플로우입니다.

---

## 주의사항: 나쁜 프로세스를 자동화하지 마세요

> "나쁜 프로세스를 자동화하면, 나쁜 결과가 더 빠르게 발생합니다."

워크플로우를 만들기 전에 반드시 해당 프로세스 자체가 올바른지 먼저 검토하세요.

- 불필요한 단계는 없는가?
- 더 간단하게 만들 수 있는가?
- 애초에 이 작업이 필요한가?

자동화는 효율을 높이는 도구이지, 나쁜 습관을 고쳐주는 마법이 아닙니다.

---

## 핵심 요약

- 워크플로우는 트리거 → 조건 → 액션의 구조로 이루어짐
- n8n은 이를 시각적 노드로 표현하여 누구나 쉽게 만들 수 있음
- 좋은 워크플로우를 만들려면 입력/처리/출력을 먼저 명확히 정의해야 함
- 자동화 전에 프로세스 자체가 올바른지 먼저 점검할 것
