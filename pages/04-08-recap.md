# 4장 핵심 정리

## 4장에서 배운 것들

4장에서는 좋은 워크플로우를 만들기 위한 실전 기술들을 배웠습니다.

---

## 핵심 개념 요약

### 워크플로우 설계 5원칙
1. 목표를 한 문장으로 정의
2. 의미 있는 노드 이름 사용
3. 왼쪽 → 오른쪽 흐름 유지
4. 조기 종료로 불필요한 처리 방지
5. 테스트 여유 공간 확보

### 조건 처리 요약

| 케이스 | 사용 노드 |
|--------|-----------|
| 참/거짓 2분기 | IF |
| 3개 이상 분기 | Switch |
| 다중 조건 | IF + AND/OR |

### 에러 처리 전략

| 전략 | 언제 사용 |
|------|-----------|
| Stop | 에러 시 즉시 중단 필요 |
| Continue | 에러를 무시하고 계속 |
| Error Output | 에러를 별도로 처리 |
| Error Trigger | 전체 워크플로우 실패 알림 |

### 디버깅 핵심 도구

| 도구 | 용도 |
|------|------|
| Pin Data | 데이터 고정으로 반복 테스트 |
| 노드 단독 실행 | 특정 노드만 테스트 |
| console.log | Code 노드 중간 확인 |
| 실행 기록 | 실제 실행 데이터 추적 |
