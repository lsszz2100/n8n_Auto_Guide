## 5장에서 배운 것들

5장에서는 단순 자동화를 넘어 AI가 판단하고 행동하는 지능형 워크플로우를 배웠습니다.

---

## 핵심 개념 요약

### AI 에이전트 4가지 유형

| 유형 | 특징 | 사용 예 |
|------|------|---------|
| Reactive | 즉각 반응, 기억 없음 | 빠른 분류 |
| Goal-based | 목표 달성 위해 다단계 행동 | 복잡한 리서치 |
| Utility-based | 최적 선택 | 채널 선택, 우선순위 결정 |
| Learning | 피드백으로 개선 | 고급 지원 봇 |

### AI 통합 핵심

| 노드 | 역할 |
|------|------|
| AI Agent | 도구 + 메모리 + 추론 |
| Basic LLM Chain | 단순 텍스트 생성 |
| Summarization Chain | 긴 텍스트 요약 |
| Q&A Chain | 문서 기반 질의응답 |

### 프롬프트 4요소

1. **명확성**: 정확히 무엇을 원하는지
2. **맥락**: 배경과 상황 정보
3. **제약**: 하지 말아야 할 것
4. **예시**: 원하는 형식의 샘플

### 메모리 유형

| 유형 | 지속성 | 사용 규모 |
|------|--------|-----------|
| Window Buffer | 임시 | 소규모 |
| Postgres | 영구 | 중대규모 |
| Redis | 임시+빠름 | 대규모 |
| Vector Store | 의미 검색 | 장기 기록 |

## 초보자가 자주 하는 실수

### 실수 1: 프롬프트를 너무 짧게 작성한다

```
나쁜 프롬프트:
  "이메일을 분류해줘"

좋은 프롬프트:
  "당신은 고객지원팀의 이메일 분류 전문가입니다.
   다음 기준으로 이메일을 분류하세요:
   - order: 주문, 결제, 영수증 관련
   - shipping: 배송, 추적, 도착 관련
   - refund: 환불, 교환, 취소 관련
   - general: 그 외 모든 문의
   
   반드시 위 4가지 중 하나만 소문자로 응답하세요."
```

### 실수 2: 메모리 없이 대화형 봇을 만든다

```
문제:
  메모리 없이 AI Agent 노드 사용
  → 고객이 "아까 말한 주문번호로..."라고 하면
  → AI가 이전 대화를 기억하지 못함

해결:
  Window Buffer Memory 추가 + Session ID 설정
  → 사용자별로 대화 기록 유지
```

### 실수 3: Temperature를 잘못 설정한다

```
데이터 추출/분류 작업:
  Temperature: 0 (항상 일관된 결과)

창의적 콘텐츠 생성:
  Temperature: 0.7~1.0 (다양한 표현)

수치 계산, 코드 생성:
  Temperature: 0 (정확도 우선)
```

## 5장 핵심 치트시트

### 언제 어떤 노드를 사용하나?

```
단순 텍스트 생성/변환:  Basic LLM Chain
문서 요약:              Summarization Chain
문서 기반 Q&A:          Q&A Chain (+ Vector Store)
자율적 판단 + 도구 사용: AI Agent
```

### AI 에이전트 비용 최적화

```
비용 절감 전략:
  1. 간단한 작업은 gpt-4o-mini 사용 (gpt-4o 대비 1/10 비용)
  2. 프롬프트 최적화로 토큰 수 줄이기
  3. 캐싱: 동일 입력 반복 시 이전 결과 재사용
  4. 필터링: AI 호출 전 명확한 케이스 먼저 처리

모델 선택 가이드:
  빠른 분류/추출:      gpt-4o-mini, claude-haiku
  복잡한 추론:         gpt-4o, claude-sonnet
  최고난도 작업:       claude-opus, o1
```

### 프롬프트 구성 템플릿

```
[역할 부여]
당신은 [역할]입니다.

[작업 설명]
다음 [입력 유형]을 받아 [작업 내용]을 수행합니다.

[출력 형식]
반드시 다음 형식으로 응답하세요:
{
  "category": "...",
  "confidence": 0.0~1.0,
  "reason": "..."
}

[제약사항]
- [하지 말아야 할 것 1]
- [하지 말아야 할 것 2]
```
