## RAG란?

**RAG(Retrieval-Augmented Generation)**는 AI가 답변할 때 관련 문서를 검색하여 그 내용을 바탕으로 답변하는 기술입니다.

**RAG가 필요한 이유:**

```
일반 AI 문제:
"우리 회사 환불 정책이 뭐야?" → AI: "모릅니다" (학습 데이터에 없음)
또는 AI: "일반적으로 30일 이내..." (할루시네이션)

RAG로 해결:
"우리 회사 환불 정책이 뭐야?"
→ 내부 문서 검색: 환불정책.pdf 관련 내용 발견
→ AI: "회사 정책에 따르면, 구매 후 7일 이내 미개봉 상품만 반품 가능합니다." ✅
```

---

## RAG 시스템 구조

```
[문서 인덱싱 파이프라인]

내부 문서 (PDF, Word, 웹페이지)
    ↓
[텍스트 추출]
    ↓
[청크(Chunk) 분할] (300-500 토큰씩)
    ↓
[임베딩 생성] (OpenAI text-embedding-3-small)
    ↓
[벡터 데이터베이스 저장] (Pinecone, Qdrant, Supabase)

─────────────────────────────────

[질의 응답 파이프라인]

사용자 질문
    ↓
[질문 임베딩 생성]
    ↓
[벡터 DB에서 유사 청크 검색] (Top 3-5)
    ↓
[검색된 청크 + 질문 → AI 프롬프트 구성]
    ↓
[AI: 컨텍스트 기반 답변 생성]
    ↓
응답
```

## n8n에서 RAG 구현

### 필요한 도구

- **벡터 DB**: Supabase (무료, Postgres 기반), Pinecone, Qdrant
- **임베딩 모델**: OpenAI text-embedding-3-small
- **AI 모델**: Claude Sonnet 또는 GPT-4o

### Supabase 설정

```sql
-- Supabase SQL 에디터에서 실행

-- pgvector 확장 활성화
create extension if not exists vector;

-- 문서 청크 테이블 생성
create table documents (
  id bigserial primary key,
  content text,
  metadata jsonb,
  embedding vector(1536)  -- OpenAI embedding 차원
);

-- 유사도 검색 함수
create function match_documents(
  query_embedding vector(1536),
  match_threshold float,
  match_count int
)
returns table (
  id bigint,
  content text,
  metadata jsonb,
  similarity float
)
language sql stable
as $$
  select
    id, content, metadata,
    1 - (documents.embedding <=> query_embedding) as similarity
  from documents
  where 1 - (documents.embedding <=> query_embedding) > match_threshold
  order by similarity desc
  limit match_count;
$$;
```

## STEP 1: 문서 인덱싱 워크플로우

```
[Manual Trigger 또는 Google Drive Trigger]
    ↓
[Google Drive: 파일 다운로드]
    ↓
[Code: 텍스트 추출 + 청크 분할]
```

```javascript
// Code 노드: 텍스트 청크 분할
const text = $input.item.json.text;
const chunkSize = 400; // 토큰 기준
const overlap = 50;    // 청크 간 오버랩

const words = text.split(' ');
const chunks = [];

for (let i = 0; i < words.length; i += chunkSize - overlap) {
  const chunk = words.slice(i, i + chunkSize).join(' ');
  if (chunk.length > 50) { // 너무 짧은 청크 제외
    chunks.push({
      content: chunk,
      chunkIndex: chunks.length,
      source: $input.item.json.fileName
    });
  }
}

return chunks.map(c => ({ json: c }));
```

```
    ↓ (각 청크마다)
[OpenAI: Create Embedding]
  Input: {{ $json.content }}
  Model: text-embedding-3-small
    ↓
[Supabase: Insert]
  Table: documents
  Data:
    content: {{ $json.content }}
    metadata: { source: {{ $json.source }}, chunkIndex: {{ $json.chunkIndex }} }
    embedding: {{ $json.embedding }}
```

## STEP 2: 질의 응답 워크플로우

```
[Telegram Trigger / Webhook]
    ↓
[OpenAI: Create Embedding]
  Input: {{ $json.userQuestion }}
    ↓
[Supabase: Execute SQL]
  Query: SELECT * FROM match_documents(
    '{{ $json.embedding }}'::vector,
    0.7,  -- 유사도 임계값
    3     -- 상위 3개 청크
  );
    ↓
[Code: 컨텍스트 구성]
```

```javascript
// 검색된 청크들을 컨텍스트로 구성
const chunks = $input.all();
const context = chunks
  .map((item, i) =>
    `[참고 자료 ${i+1}]\n${item.json.content}`
  )
  .join('\n\n');

return [{
  json: {
    context,
    question: $node["Telegram Trigger"].json.message.text
  }
}];
```

```
    ↓
[Claude Sonnet: 답변 생성]
  System: |
    너는 회사 내부 지식 베이스를 기반으로 답변하는 어시스턴트야.
    반드시 제공된 참고 자료에 근거해서만 답변해.
    참고 자료에 없는 내용은 "죄송합니다, 해당 내용은 확인되지 않습니다"라고 답해.
  User: |
    참고 자료:
    {{ $json.context }}

    질문: {{ $json.question }}
    ↓
[Telegram / Webhook: 응답 발송]
```

## 품질 향상 기법

### 청크 크기 최적화

| 사용 사례 | 청크 크기 | 오버랩 |
|-----------|-----------|--------|
| 짧은 FAQ | 200 토큰 | 20 |
| 기술 문서 | 400 토큰 | 50 |
| 법률 계약서 | 600 토큰 | 100 |

### 메타데이터 활용

```javascript
// 파일 종류, 날짜, 부서 등 메타데이터 저장
metadata: {
  source: "환불정책_v3.pdf",
  department: "CS",
  lastUpdated: "2026-04-01",
  category: "정책"
}

// 검색 시 메타데이터 필터링
WHERE metadata->>'category' = '정책'
```

## 핵심 요약

- RAG = 문서 검색 + AI 생성을 결합한 신뢰할 수 있는 AI 응답
- 인덱싱 파이프라인: 문서 → 청크 → 임베딩 → 벡터 DB
- 질의 파이프라인: 질문 임베딩 → 유사 청크 검색 → AI 컨텍스트 기반 답변
- Supabase + pgvector로 무료로 시작 가능
- 메타데이터 필터링으로 정확도 향상
