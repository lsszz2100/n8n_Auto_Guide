# 레슨 7.5: RAG 시스템 구축하기

---

## RAG란?

**RAG(Retrieval-Augmented Generation)**는 AI가 답변할 때 관련 문서를 검색하여 그 내용을 바탕으로 답변하는 기술입니다.

**RAG가 필요한 이유:**

```
일반 AI 문제:
"우리 회사 환불 정책이 뭐야?" → AI: "모릅니다" (학습 데이터에 없음)
또는 AI: "일반적으로 30일 이내..." (할루시네이션)

RAG로 해결:
"우리 회사 환불 정책이 뭐야?"
→ 내부 문서 검색: 환불정책.pdf 관련 내용 발견
→ AI: "회사 정책에 따르면, 구매 후 7일 이내 미개봉 상품만 반품 가능합니다." ✅
```

---

## RAG 시스템 구조

```
[문서 인덱싱 파이프라인]

내부 문서 (PDF, Word, 웹페이지)
    ↓
[텍스트 추출]
    ↓
[청크(Chunk) 분할] (300-500 토큰씩)
    ↓
[임베딩 생성] (OpenAI text-embedding-3-small)
    ↓
[벡터 데이터베이스 저장] (Pinecone, Qdrant, Supabase)

─────────────────────────────────

[질의 응답 파이프라인]

사용자 질문
    ↓
[질문 임베딩 생성]
    ↓
[벡터 DB에서 유사 청크 검색] (Top 3-5)
    ↓
[검색된 청크 + 질문 → AI 프롬프트 구성]
    ↓
[AI: 컨텍스트 기반 답변 생성]
    ↓
응답
```

## n8n에서 RAG 구현

### 필요한 도구

- **벡터 DB**: Supabase (무료, Postgres 기반), Pinecone, Qdrant
- **임베딩 모델**: OpenAI text-embedding-3-small
- **AI 모델**: Claude Sonnet 또는 GPT-4o

### Supabase 설정

```sql
-- Supabase SQL 에디터에서 실행

-- pgvector 확장 활성화
create extension if not exists vector;

-- 문서 청크 테이블 생성
create table documents (
  id bigserial primary key,
  content text,
  metadata jsonb,
  embedding vector(1536)  -- OpenAI embedding 차원
);

-- 유사도 검색 함수
create function match_documents(
  query_embedding vector(1536),
  match_threshold float,
  match_count int
)
returns table (
  id bigint,
  content text,
  metadata jsonb,
  similarity float
)
language sql stable
as $$
  select
    id, content, metadata,
    1 - (documents.embedding <=> query_embedding) as similarity
  from documents
  where 1 - (documents.embedding <=> query_embedding) > match_threshold
  order by similarity desc
  limit match_count;
$$;
```

## STEP 1: 문서 인덱싱 워크플로우

```
[Manual Trigger 또는 Google Drive Trigger]
    ↓
[Google Drive: 파일 다운로드]
    ↓
[Code: 텍스트 추출 + 청크 분할]
```

```javascript
// Code 노드: 텍스트 청크 분할
const text = $input.item.json.text;
const chunkSize = 400; // 토큰 기준
const overlap = 50;    // 청크 간 오버랩

const words = text.split(' ');
const chunks = [];

for (let i = 0; i < words.length; i += chunkSize - overlap) {
  const chunk = words.slice(i, i + chunkSize).join(' ');
  if (chunk.length > 50) { // 너무 짧은 청크 제외
    chunks.push({
      content: chunk,
      chunkIndex: chunks.length,
      source: $input.item.json.fileName
    });
  }
}

return chunks.map(c => ({ json: c }));
```

```
    ↓ (각 청크마다)
[OpenAI: Create Embedding]
  Input: {{ $json.content }}
  Model: text-embedding-3-small
    ↓
[Supabase: Insert]
  Table: documents
  Data:
    content: {{ $json.content }}
    metadata: { source: {{ $json.source }}, chunkIndex: {{ $json.chunkIndex }} }
    embedding: {{ $json.embedding }}
```

### 예제 1: 회사 내부 문서 검색 봇 (Slack에서 질문 → 문서 검색 → 답변)

직원이 Slack에서 질문하면 사내 문서(정책, 매뉴얼, 가이드라인)를 검색하여 근거 있는 답변을 제공합니다. 답변 출처 문서명도 함께 표시합니다.

워크플로우 구조:

```
[Slack Trigger: 봇 멘션 감지]
  (@내부봇 질문 내용)
    ↓
[Code: 질문 텍스트 추출 및 전처리]
    ↓
[OpenAI: 질문 임베딩 생성]
  모델: text-embedding-3-small
    ↓
[Supabase: 유사 문서 청크 검색]
  함수: match_documents(embedding, 0.65, 5)
  메타데이터 필터: department 기반 (HR, 법무, IT 등)
    ↓
[Code: 컨텍스트 구성 + 출처 목록 정리]
    ↓
[Claude Sonnet: 답변 생성]
    ↓
[Slack: 스레드에 답변 발송 (출처 문서 포함)]
```

문서 검색 결과 컨텍스트 구성 코드:

```javascript
const chunks = $input.all();

if (chunks.length === 0) {
  return [{
    json: {
      hasContext: false,
      context: "",
      sources: [],
      question: $node["Code"].json.question
    }
  }];
}

// 중복 문서 제거 후 출처 목록 구성
const seenSources = new Set();
const sources = [];
const contextParts = [];

chunks.forEach((chunk, idx) => {
  const { content, metadata, similarity } = chunk.json;
  const sourceName = metadata.source;

  contextParts.push(`[참고 자료 ${idx + 1}] (출처: ${sourceName})\n${content}`);

  if (!seenSources.has(sourceName)) {
    seenSources.add(sourceName);
    sources.push({
      name: sourceName,
      department: metadata.department,
      lastUpdated: metadata.lastUpdated,
      similarity: (similarity * 100).toFixed(1)
    });
  }
});

return [{
  json: {
    hasContext: true,
    context: contextParts.join('\n\n'),
    sources,
    question: $node["Code"].json.question,
    channelId: $node["Code"].json.channelId,
    threadTs: $node["Code"].json.threadTs
  }
}];
```

### 예제 2: 상품 매뉴얼 기반 고객 Q&A 자동화

고객이 제품 사용법을 문의하면 해당 제품의 공식 매뉴얼에서 관련 내용을 찾아 정확한 답변을 제공합니다. 제품별로 다른 매뉴얼을 관리합니다.

워크플로우 구조:

```
[Webhook: 고객 문의 수신]
  (제품 ID + 질문 포함)
    ↓
[Code: 제품 ID로 해당 매뉴얼 카테고리 결정]
    ↓
[OpenAI: 질문 임베딩 생성]
    ↓
[Supabase: 제품별 매뉴얼 검색]
  WHERE metadata->>'product_id' = '{{ $json.productId }}'
    ↓
[IF: 검색 결과 있음?]
  True →
    [Claude Sonnet: 매뉴얼 기반 답변 생성]
    ↓
    [Code: 답변 + 관련 매뉴얼 페이지 번호 포함]
    ↓
    [Webhook Response: 답변 반환]
  False →
    [AI: "매뉴얼에 없는 내용" 안내 + 고객센터 연결]
```

제품 매뉴얼 인덱싱 설정 (초기 1회 실행):

```
[Manual Trigger]
    ↓
[Google Drive: 매뉴얼 폴더에서 파일 목록 조회]
  폴더 ID: MANUAL_FOLDER_ID
    ↓
[Loop: 각 파일 처리]
    ↓
[HTTP Request: 파일 내용 다운로드]
    ↓
[Code: 텍스트 청크 분할 + 메타데이터 추출]
```

```javascript
// 파일명에서 제품 정보 추출
// 파일명 규칙: "PROD-001_에어컨_매뉴얼_v2.pdf"
const fileName = $json.name;
const parts = fileName.split('_');
const productId = parts[0];           // PROD-001
const productName = parts[1];         // 에어컨
const version = parts[parts.length - 1].replace('.pdf', '');  // v2

const text = $json.extractedText;
const chunkSize = 350;
const overlap = 40;
const words = text.split(/\s+/);
const chunks = [];

for (let i = 0; i < words.length; i += chunkSize - overlap) {
  const chunk = words.slice(i, i + chunkSize).join(' ');
  if (chunk.length > 80) {
    // 페이지 번호 추정 (300단어 ≈ 1페이지)
    const estimatedPage = Math.floor(i / 300) + 1;
    chunks.push({
      content: chunk,
      metadata: {
        product_id: productId,
        product_name: productName,
        source: fileName,
        estimated_page: estimatedPage,
        version
      }
    });
  }
}

return chunks.map(c => ({ json: c }));
```

매뉴얼 기반 답변 생성 프롬프트:

```
[Claude Sonnet]

User: |

  {{ $json.context }}
```

### 예제 3: 법률 판례 검색 시스템 (질문 → 유사 판례 3개 + 요약)

법률 관련 질문을 입력하면 기존에 인덱싱된 판례 데이터베이스에서 유사한 판례 3개를 검색하고, 각 판례의 핵심 내용을 요약하여 제공합니다.

워크플로우 구조:

```
[Webhook: 법률 질문 수신]
    ↓
[OpenAI: 질문 임베딩 생성]
    ↓
[Supabase: 유사 판례 검색]
  함수: match_documents(embedding, 0.60, 3)
  WHERE metadata->>'doc_type' = 'precedent'
    ↓
[Code: 판례 목록 구성]
    ↓
[OpenAI: 각 판례 개별 요약 생성] (판례별 루프)
    ↓
[Code: 최종 응답 조합]
    ↓
[Webhook Response: 판례 3개 + 요약 반환]
```

판례 데이터 인덱싱 구조:

```javascript

// 판례 청크 크기: 600 토큰 (법률 문서 특성상 더 큰 청크)
```

최종 응답 조합 코드:

```javascript
const summaries = $input.all().map(item => item.json);
const userQuestion = $node["Webhook"].json.question;

const formattedResponse = {
  question: userQuestion,
  searchedAt: new Date().toISOString(),
  disclaimer: "이 결과는 참고용 정보이며, 실제 법률 판단은 전문 변호사와 상담하세요.",
  precedents: summaries.map((p, idx) => ({
    rank: idx + 1,
    caseNumber: p.caseNumber,
    court: p.court,
    decisionDate: p.decisionDate,
    coreIssue: p.coreIssue,
    decision: p.decision,
    legalPrinciple: p.legalPrinciple,
    relevanceNote: p.relevanceNote
  }))
};

// 유사 판례가 없는 경우 처리
if (summaries.length === 0) {
  formattedResponse.message = "입력하신 질문과 유사한 판례를 찾지 못했습니다. 질문을 더 구체적으로 입력하거나 법률 전문가에게 문의하세요.";
}

return [{ json: formattedResponse }];
```
