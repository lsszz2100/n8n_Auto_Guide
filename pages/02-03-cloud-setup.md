# 레슨 2.3: n8n Cloud 시작하기

---

## n8n Cloud 계정 만들기

n8n Cloud는 설치 없이 5분 안에 시작할 수 있는 가장 빠른 방법입니다.

### 단계별 안내

**1단계: 회원가입**
1. 브라우저에서 `app.n8n.cloud` 접속
2. "Get started for free" 클릭
3. 이메일, 이름, 비밀번호 입력
4. 이메일 인증 완료

**2단계: 플랜 선택**
- 처음에는 **14일 무료 체험**으로 시작
- 체험 기간 동안 Pro 플랜의 모든 기능 사용 가능
- 신용카드 없이 시작 가능 (체험 기간 한정)

**3단계: 인스턴스 생성**
- 지역 선택: 한국에서 사용한다면 **Asia Pacific** 선택 (지연 시간 최소화)
- 인스턴스 이름 설정
- "Create instance" 클릭

약 1~2분 후 n8n 인스턴스가 생성됩니다.

## n8n Cloud 주요 설정

### 1. 프로필 설정
우측 상단 아이콘 → Settings:
- 이름, 이메일 변경
- 비밀번호 변경
- 2단계 인증 설정 (보안을 위해 강력 권장!)

### 2. 타임존 설정
워크플로우에서 시간 기반 트리거를 사용한다면 타임존을 반드시 올바르게 설정해야 합니다.

Settings → General → Timezone: **Asia/Seoul**

### 3. 이메일 알림 설정
워크플로우가 실패했을 때 이메일로 알림을 받도록 설정할 수 있습니다.

Settings → Notifications → Workflow failures

## 요금제 관리

### 현재 사용량 확인
대시보드 우측 상단에서 이번 달 실행 횟수와 남은 횟수를 확인할 수 있습니다.

### 실행 횟수 절약 팁
1. **불필요한 트리거 줄이기**: Polling 방식 트리거는 실행 횟수를 더 빨리 소비합니다
2. **Webhook 사용**: Polling 대신 Webhook으로 교체하면 실행 횟수 절약
3. **에러 줄이기**: 실패한 실행도 횟수에 포함되므로 에러를 최소화

---

## n8n Cloud의 자동 백업

Cloud 버전은 n8n이 자동으로 백업을 관리합니다:
- 워크플로우는 자동 저장
- 실행 기록 30일 보관
- 장애 시 n8n 팀이 복구

---

## 커스텀 도메인 연결 (Pro 이상)

기본 URL 대신 내 도메인을 사용할 수 있습니다:

`n8n.mycompany.com` 형식으로 설정 가능

Settings → Custom Domain에서 설정합니다.

## 문제 해결

### 이메일 인증 메일이 안 왔을 때
1. 스팸 폴더 확인
2. "Resend verification email" 클릭
3. 그래도 안 온다면 다른 이메일 주소로 재시도

### 로그인이 안 될 때
1. 비밀번호 재설정 진행
2. 브라우저 캐시/쿠키 삭제 후 재시도

### 속도가 느릴 때
- 가까운 지역의 서버를 선택했는지 확인
- 인스턴스를 삭제하고 Asia Pacific 지역으로 재생성

---

## 핵심 요약

- n8n Cloud는 5분 안에 시작 가능한 가장 쉬운 방법
- 14일 무료 체험 (신용카드 불필요)
- 타임존을 Asia/Seoul로 반드시 설정
- 실행 횟수를 절약하려면 Polling 대신 Webhook 사용
- Cloud 버전은 백업과 유지보수를 n8n이 담당
